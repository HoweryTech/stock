# 价值质量策略筛选

本文档定义第一阶段价值质量策略的可执行筛选规则。

价值质量策略不直接给出买入结论，只生成候选股。候选股进入观察池后，仍需要补充估值、行业景气、反证风险、买入价、止损价和仓位计划，才能进入交易计划风控校验。

## 1. 输入

默认读取：

```text
data/processed/financial_metrics.csv
```

该文件由 `tools/import_financial_metrics.py` 生成，字段标准见 [财务核心指标导入](./财务核心指标导入.md)。

同一股票存在多个报告期时，筛选器只使用最新报告期。

## 2. 配置

筛选阈值来自 `config/investment-profile.example.yaml`：

```yaml
strategies:
  value_quality:
    screening:
      min_roe: 4.0
      min_roa: 1.0
      min_operating_cash_flow: 0
      max_debt_ratio: 75.0
      min_revenue_growth_yoy: 0.0
      min_deducted_net_profit_growth_yoy: 0.0
      min_gross_margin: 0.0
      max_candidates: 20
```

第一版采用偏保守阈值：要求盈利能力、现金流、负债率、营收增长和扣非利润增长同时过线。

## 3. 筛选规则

候选股必须同时满足：

- ROE 不低于阈值。
- ROA 不低于阈值。
- 经营现金流净额不低于阈值。
- 资产负债率不高于阈值。
- 营收同比不低于阈值。
- 扣非净利润同比不低于阈值。
- 毛利率不低于阈值。

任一指标缺失时，视为无法验证并排除。

## 4. 输出

默认输出候选股：

```text
data/processed/value_quality_candidates.csv
```

默认输出元数据：

```text
data/metadata/value_quality_candidates.json
```

候选股 CSV 包含：

```text
code,report_period,score,roe,roa,gross_margin,net_margin,debt_ratio,operating_cash_flow,revenue_growth_yoy,net_profit_growth_yoy,deducted_net_profit_growth_yoy,eps,reasons,risks
```

其中：

- `score` 是可解释的基础排序分，不是收益预测。
- `reasons` 记录入选理由。
- `risks` 记录仍需人工确认的风险提示。
- 元数据记录输入数量、候选数量、排除数量和排除原因。

## 5. 命令

```bash
python3 tools/screen_value_quality.py \
  --profile config/investment-profile.example.yaml \
  --financial-metrics data/processed/financial_metrics.csv \
  --output data/processed/value_quality_candidates.csv \
  --metadata-output data/metadata/value_quality_candidates.json
```

使用示例数据验证：

```bash
python3 tools/screen_value_quality.py \
  --financial-metrics samples/financial_metrics.sample.csv \
  --output /tmp/value_quality_candidates.csv \
  --metadata-output /tmp/value_quality_candidates.json
```
