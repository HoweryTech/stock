# 多策略候选池合并

本文档定义第一阶段统一候选池的合并规则。

趋势强度、价值质量等策略先各自独立筛选，随后再进入统一候选池。这样可以避免策略之间互相污染，同时让同一只股票的多策略证据集中呈现。

## 1. 输入

默认读取：

```text
data/processed/trend_candidates.csv
data/processed/value_quality_candidates.csv
data/processed/event_catalyst_candidates.csv
data/processed/tradable_universe.csv
data/processed/industry_strength_factors.csv
```

后续新增事件催化、估值分位、行业强度等策略时，也应先生成独立候选文件，再接入统一候选池。
事件催化候选是可选输入；未提供时不影响趋势和价值质量候选合并。
如果传入可交易股票池，合并工具会补充股票名称、行业和流动性证据。
如果传入行业强度因子，合并工具会补充行业强度分和行业强度证据。

## 2. 合并规则

合并键为 `code`。

同一股票来自多个策略时：

- `primary_strategy` 标记为 `multi_strategy`。
- `strategies` 记录全部策略来源。
- `reasons` 按策略来源加前缀保留。
- `risks` 按策略来源加前缀保留。
- `trend_score` 和 `value_quality_score` 分别保留，不强行解释为同一量纲。
- `event_score`、`event_date` 和 `event_type` 保留结构化事件证据。
- `name`、`industry` 优先来自传入的股票池。
- `exchange` 来自传入的股票池。
- `board` 根据交易所和股票代码推断，用于区分沪市主板、深市主板、科创板、创业板和北交所。
- `liquidity_score` 基于趋势窗口平均成交额或股票池平均成交额生成，只用于观察池排序辅助和门禁解释。
- `industry_strength_score` 来自行业强度因子，并以小权重加入 `combined_score`。
- `strategy_confluence_score` 和 `strategy_confluence_evidence` 解释多策略共振。
- `data_quality_score`、`data_quality_status` 和 `data_quality_evidence` 解释证据完整性。
- `risk_penalty_score` 和 `risk_penalty_evidence` 解释风险扣分。

统一排序使用 `combined_score`：

```text
策略共振分 + 趋势分 + 价值质量分 + 事件分 + 行业强度分 * 0.2 + 流动性分 * 0.1 + 数据质量分 + 风险扣分
```

其中策略共振分等于策略数量乘以 100；数据质量分最高 20；风险扣分为 0 或负数。该分数只用于观察池排序，不是收益预测，也不能替代交易计划。

## 3. 输出

默认输出候选池：

```text
data/processed/candidate_pool.csv
```

默认输出元数据：

```text
data/metadata/candidate_pool.json
```

候选池 CSV 包含：

```text
code,name,industry,exchange,board,strategies,strategy_count,combined_score,primary_strategy,strategy_confluence_score,strategy_confluence_evidence,trend_score,value_quality_score,event_score,event_date,event_type,liquidity_score,liquidity_evidence,industry_strength_score,industry_strength_evidence,data_quality_score,data_quality_status,data_quality_evidence,risk_penalty_score,risk_penalty_evidence,technical_health_score,technical_health_status,technical_health_evidence,technical_risk_flags,trade_date,report_period,reasons,risks
```

组合适配步骤会在上述字段后追加 `portfolio_fit_status`、`portfolio_fit_action` 和买入后仓位测算字段，不改变 `combined_score`。

`technical_health_score` 来自日线、周线、月线的 MACD、BOLL、RSI、KDJ、ATR 和量比综合判断；状态为 `strong`、`watch`、`weak`、`blocked` 或 `insufficient`。技术健康分是门禁/扣分型因子，不能单独构成买入理由；`blocked` 会阻断进入买入计划。

元数据包含每个策略输入数量、合并后候选数量、多策略候选数量，以及技术指标覆盖数量和技术阻断数量。

## 4. 命令

```bash
python3 tools/merge_candidate_pool.py \
  --trend-candidates data/processed/trend_candidates.csv \
  --value-quality-candidates data/processed/value_quality_candidates.csv \
  --event-candidates data/processed/event_catalyst_candidates.csv \
  --universe data/processed/tradable_universe.csv \
  --industry-strength data/processed/industry_strength_factors.csv \
  --technical-indicators data/metadata/technical-indicators.json \
  --output data/processed/candidate_pool.csv \
  --metadata-output data/metadata/candidate_pool.json
```
