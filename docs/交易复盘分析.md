# 交易复盘分析

本文档定义如何从复盘记录生成策略迭代分析。

复盘分析不预测未来收益，也不自动修改策略参数。它只汇总已经发生的交易结果，帮助识别哪些策略、错误标签和复盘质量问题需要关注。

## 1. 输入

默认读取：

```text
reviews/*.yaml
```

## 2. 输出

默认输出：

```text
reports/review-analysis.md
```

可选输出 JSON：

```text
data/metadata/review-analysis.json
```

真实分析报告默认不提交到 Git。

## 3. 统计内容

- 总复盘数量。
- 胜率。
- 平均单笔收益率。
- 组合收益贡献合计。
- 按策略汇总胜率、收益和组合贡献。
- 结果分类分布。
- 错误标签分布。
- 复盘质量分布。
- 冷静期例外交易数量、平均收益率和组合贡献。
- 策略健康例外交易数量、平均收益率和组合贡献。

## 4. 纪律例外

复盘分析会单独列出纪律例外交易。这里的纪律例外包括：

- 冷静期内仍然买入。
- 策略健康检查已暂停新开仓，但仍然买入。

这类交易不是直接禁止统计，而是必须单独归因：如果例外交易持续亏损，说明破例流程在破坏纪律；如果例外交易确实改善结果，也需要复查原规则是否过严。

## 5. 命令

```bash
python3 tools/analyze_trade_reviews.py \
  --reviews reviews/*.yaml \
  --output reports/review-analysis.md \
  --json-output data/metadata/review-analysis.json
```

如果只想看 JSON：

```bash
python3 tools/analyze_trade_reviews.py --json
```
