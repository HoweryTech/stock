# 卖出执行检查

本文档定义卖出执行记录进入复盘前的检查。

卖出执行检查用于回答：卖出记录是否字段完整、是否仍符合退出计划检查结论、是否具备必要的 confirmed 人工确认快照、卖出价格和仓位是否越过风控边界。

## 1. 输入

默认读取：

```text
exit-executions/*.yaml
```

## 2. 检查结论

- `pass`：可以进入卖出执行生成复盘流程。
- `needs_review`：可以生成复盘草稿，但需要在复盘中解释偏差。
- `blocked`：默认不能生成复盘，必须先修正卖出执行记录或补齐确认快照。

## 3. 阻断项

会阻断的情况包括：

- 缺少卖出执行编号、来源退出计划、来源持仓、来源交易计划、卖出日期、卖出价格或卖出仓位。
- 退出计划检查结论不是 `pass` 或 `needs_review`。
- `needs_review` 或真实卖出缺少 confirmed 人工确认快照。
- 卖出价低于最低可接受退出价。
- 卖出仓位高于当前持仓。

## 4. 命令

```bash
python3 tools/check_exit_execution.py \
  --exit-execution exit-executions/EXITEXEC-示例.yaml
```

输出 JSON：

```bash
python3 tools/check_exit_execution.py \
  --exit-execution exit-executions/EXITEXEC-示例.yaml \
  --json
```
