# 复盘冷静期检查

本文档定义基于复盘结果的连续亏损冷静期检查。

冷静期检查不判断下一笔交易是否一定亏损，只根据投资体系配置中的连续亏损阈值，提醒是否应该暂停新开仓或暂停某个策略的新开仓。

## 1. 输入

默认读取：

```text
config/investment-profile.example.yaml
reviews/*.yaml
```

使用配置：

```yaml
risk:
  losing_streak_cooldown:
    enabled: true
    consecutive_losing_trades: 3
    cooldown_trading_days: 5
```

## 2. 结论

- `normal`：没有触发冷静期。
- `cooldown_required`：整体或某策略连续亏损达到阈值。

## 3. 命令

```bash
python3 tools/check_review_cooldown.py \
  --reviews reviews/*.yaml \
  --output data/metadata/review-cooldown.json
```

输出 JSON：

```bash
python3 tools/check_review_cooldown.py --json
```

## 4. 使用纪律

当结论为 `cooldown_required` 时，不应继续生成新的买入执行记录。至少需要完成：

- 补全最近亏损交易的复盘。
- 更新交易复盘分析。
- 明确冷静期结束条件。
- 如需例外交易，必须手工记录原因。
