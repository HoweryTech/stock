# 事件催化策略筛选

本文档定义第一版事件催化候选股筛选规则。

事件催化策略只处理结构化事件 CSV，不直接抓取公告，也不替代公告原文核验。事件候选只代表进入观察池，不能直接作为买入建议。

## 1. 输入

默认读取：

```text
data/processed/event_catalyst_events.csv
config/investment-profile.example.yaml
```

样例文件：

```text
samples/event_catalyst_events.sample.csv
```

事件 CSV 字段：

```text
event_date,code,event_type,title,expected_impact,impact_score,confidence,counter_evidence,risk_disclosure,source,updated_at
```

## 2. 筛选规则

候选必须满足：

- 事件类型在 `strategies.event_catalyst.supported_event_types` 中。
- 不是监管问询、减持、解禁、诉讼、财务更正等风险事件。
- `impact_score` 不低于阈值。
- `confidence` 不低于阈值。
- 有预期影响、反证和风险披露。

风险事件不会进入事件催化候选池，应进入风险复核或公告风险流程。

## 3. 输出

默认输出：

```text
data/processed/event_catalyst_candidates.csv
data/metadata/event_catalyst_candidates.json
```

候选 CSV 包含：

```text
code,event_date,event_type,score,title,expected_impact,counter_evidence,risk_disclosure,reasons,risks
```

## 4. 命令

```bash
python3 tools/screen_event_catalyst.py \
  --profile config/investment-profile.example.yaml \
  --events data/processed/event_catalyst_events.csv \
  --output data/processed/event_catalyst_candidates.csv \
  --metadata-output data/metadata/event_catalyst_candidates.json
```

使用样例数据验证：

```bash
python3 tools/screen_event_catalyst.py \
  --events samples/event_catalyst_events.sample.csv \
  --output /tmp/event_catalyst_candidates.csv \
  --metadata-output /tmp/event_catalyst_candidates.json
```
