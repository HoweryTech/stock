# 候选池组合适配

本文档定义候选股进入交易计划前的组合适配检查。

组合适配不改变候选股的选股分数，也不输出买入建议。它只回答：在当前持仓、行业暴露、总仓位和策略健康约束下，这个候选是否适合继续推进到交易计划草稿。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
config/investment-profile.example.yaml
positions/*.yaml
data/metadata/strategy-health.json
```

`positions/*.yaml` 和 `strategy-health.json` 可按命令参数替换。若没有读取到持仓，工具会按空仓测算，并在证据中注明。

## 2. 输出

默认输出：

```text
data/processed/candidate_pool.portfolio_fit.csv
data/metadata/candidate_portfolio_fit.json
```

输出会在候选池中新增：

```text
portfolio_fit_status
portfolio_fit_action
portfolio_fit_evidence
current_stock_position_pct
current_industry_position_pct
current_total_position_pct
expected_stock_position_pct_after_buy
expected_industry_position_pct_after_buy
expected_total_position_pct_after_buy
```

## 3. 状态

- `ready_for_plan`：组合仓位、行业暴露和策略健康检查未发现阻断，可继续准备交易计划。
- `watch`：没有硬阻断，但存在同行业暴露、接近仓位上限或策略需复核等提醒，应继续观察或人工复核。
- `deferred_by_portfolio`：已有同票持仓、买入后超过单票/行业/总仓位上限，或策略健康已暂停新开仓，应暂缓进入交易计划。

## 4. 命令

```bash
python3 tools/apply_candidate_portfolio_fit.py \
  --candidates data/processed/candidate_pool.csv \
  --positions 'positions/*.yaml' \
  --strategy-health data/metadata/strategy-health.json \
  --planned-position-pct 5 \
  --output data/processed/candidate_pool.portfolio_fit.csv \
  --metadata-output data/metadata/candidate_portfolio_fit.json
```

若希望观察池报告直接显示组合适配状态，可以用组合适配后的候选池生成报告：

```bash
python3 tools/generate_watchlist_report.py \
  --candidates data/processed/candidate_pool.portfolio_fit.csv \
  --output reports/watchlist.md
```

## 5. 决策边界

`ready_for_plan` 只表示可以继续准备交易计划，不代表可以买入。

真实交易前仍必须补齐买入价、止损价、仓位、失效条件，并通过交易计划质量检查、风控校验和人工确认。
