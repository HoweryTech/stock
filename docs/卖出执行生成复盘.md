# 卖出执行生成复盘

本文档定义从卖出执行记录生成交易复盘草稿的流程。

该工具只搬运已经发生的执行事实，并计算收益归因；复盘结论、经验教训和下一步动作仍需要人工填写或确认。

## 1. 输入

默认读取：

```text
exit-executions/*.yaml
templates/trade-review.example.yaml
```

工具会先运行卖出执行检查。检查结论为 `blocked` 时，默认不能生成复盘草稿；只有明确处于修正历史记录的工作流，才使用 `--allow-blocked-exit-execution` 继续生成，并把检查结果写入 `exit_execution_check_snapshot`。

工具会从卖出执行记录中提取：

- 来源交易计划、来源持仓、来源退出计划、来源卖出执行。
- 入场日期、入场价、退出日期、退出价、退出仓位。
- 退出原因和是否符合原计划。
- 单笔收益率和组合收益贡献。
- 入场时是否处于冷静期或策略暂停期，以及是否属于例外交易。
- 策略配置版本快照，用于确认该笔交易使用的是哪一版投资体系。

## 2. 结果分类

默认沿用交易复盘规则：

- 符合计划且收益为正：`strategy_profit`。
- 符合计划且收益为负：`strategy_loss`。
- 不符合计划但收益为正：`execution_error_profit`。
- 不符合计划且收益为负：`execution_error_loss`。

## 3. 纪律例外

如果卖出执行快照中包含原买入执行记录，工具会自动写入：

- `discipline.cooldown_conclusion_at_entry`
- `discipline.strategy_health_conclusion_at_entry`
- `discipline.was_cooldown_exception`
- `discipline.was_strategy_health_exception`
- `discipline.exception_reason`

这些字段用于后续复盘分析，判断破例交易是否真的改善结果。

复盘草稿会把卖出执行记录或原交易计划中的 `strategy_config_snapshot` 提到顶层，同时保留 `trade_plan_snapshot`、`exit_plan_snapshot` 和 `exit_execution_snapshot` 作为完整证据链。
同时会保留 `exit_execution_check_snapshot`，用于复盘时确认该卖出执行是否存在价格、仓位或人工确认缺口。

## 4. 命令

```bash
python3 tools/new_trade_review_from_exit_execution.py \
  --exit-execution exit-executions/EXITEXEC-示例.yaml \
  --output reviews/TR-示例.yaml
```

补充初始复盘内容：

```bash
python3 tools/new_trade_review_from_exit_execution.py \
  --exit-execution exit-executions/EXITEXEC-示例.yaml \
  --lesson "止损执行及时。" \
  --next-action "复查该策略在弱市中的胜率。"
```

修正历史记录时允许从被阻断的卖出执行生成复盘草稿：

```bash
python3 tools/new_trade_review_from_exit_execution.py \
  --exit-execution exit-executions/EXITEXEC-示例.yaml \
  --allow-blocked-exit-execution \
  --output reviews/TR-修正示例.yaml
```
