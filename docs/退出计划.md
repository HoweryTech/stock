# 退出计划

本文档定义持仓退出前的结构化计划。

退出计划不是交易复盘。它发生在卖出前，用于记录为什么卖、卖多少、按什么价格或条件卖、是否符合原计划。卖出完成后，再进入交易复盘。

## 1. 输入

默认读取：

```text
positions/*.yaml
templates/exit-plan.example.yaml
```

可选读取：

```text
data/metadata/*.daily-check.json
```

如果不传日检 JSON，工具会根据当前持仓现场执行一次持仓风险检查。

## 2. 退出类型

第一版支持：

- `stop_loss`：触发止损。
- `take_profit`：止盈或达到目标区。
- `thesis_invalidated`：买入假设失效。
- `risk_reduction`：组合降风险。

## 3. 命令

从持仓日检生成：

```bash
python3 tools/new_exit_plan.py \
  --position positions/POS-示例.yaml \
  --daily-check data/metadata/POS-示例.daily-check.json \
  --output exit-plans/EXIT-示例.yaml
```

手工指定退出类型：

```bash
python3 tools/new_exit_plan.py \
  --position positions/POS-示例.yaml \
  --exit-type risk_reduction \
  --exit-reason "组合需要降低权益仓位。" \
  --evidence "市场环境转弱。" \
  --risk-if-hold "继续持有可能放大组合回撤。" \
  --exit-position-pct 3
```

## 4. 输出

默认输出：

```text
exit-plans/EXIT-YYYYMMDD-HHMMSS.yaml
```

真实退出计划默认不提交到 Git。
