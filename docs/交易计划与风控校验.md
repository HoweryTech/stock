# 交易计划与风控校验

本文档定义买入前必须填写的交易计划，以及系统必须执行的风控校验。

目标不是让交易流程变复杂，而是避免在没有证据、没有止损、没有仓位约束的情况下临时交易。

## 1. 核心原则

每一笔交易在买入前必须回答：

1. 为什么买。
2. 依据哪个策略买。
3. 证据是什么。
4. 反证和风险是什么。
5. 买错最多亏多少。
6. 什么情况下证明这笔交易失效。
7. 是否违反投资体系配置。

没有交易计划，就不进入买入流程。

## 2. 交易计划生命周期

```text
草稿
  -> 待风控校验
  -> 风控通过
  -> 人工确认
  -> 已执行
  -> 持仓跟踪
  -> 已退出
  -> 已复盘
```

状态说明：

| 状态 | 含义 |
| --- | --- |
| 草稿 | 初步记录交易想法，还不能交易 |
| 待风控校验 | 必填字段已完成，等待规则校验 |
| 风控通过 | 未触发阻断规则，可进入人工确认 |
| 人工确认 | 用户确认接受计划和最大亏损 |
| 已执行 | 已发生真实或模拟成交 |
| 持仓跟踪 | 持仓期间持续监控风险 |
| 已退出 | 已全部卖出 |
| 已复盘 | 已完成交易归因 |

## 3. 买入前必填字段

交易计划必须包含以下字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 股票代码 | 是 | A 股证券代码 |
| 股票名称 | 是 | 标的名称 |
| 策略来源 | 是 | 价值质量、趋势强度、事件催化等 |
| 买入理由 | 是 | 用自然语言描述交易假设 |
| 关键证据 | 是 | 行情、财务、公告、行业或回测证据 |
| 反证和风险 | 是 | 与买入理由相反或削弱逻辑的证据 |
| 计划买入价 | 是 | 理想买入价格 |
| 最大可接受买入价 | 是 | 超过该价格禁止追入 |
| 止损价 | 是 | 判断错误时退出价格或条件 |
| 计划仓位 | 是 | 占总资产比例 |
| 单笔最大亏损 | 是 | 按买入价、止损价和仓位计算 |
| 卖出条件 | 是 | 止盈、趋势破坏、逻辑兑现等 |
| 失效条件 | 是 | 买入假设被证伪的条件 |
| 策略配置快照 | 否 | 记录生成计划时使用的 `version_id` 和 `profile_hash`，用于后续复盘和回测归因 |
| 观察项 | 否 | 持仓后需要持续观察的事项 |

## 4. 风控校验分类

风控校验分为三类。

### 4.1 阻断

触发后禁止进入人工确认。

包括：

- 无策略来源。
- 无止损价。
- 无关键证据。
- 无反证和风险。
- 单笔最大亏损超过配置阈值。
- 单票仓位超过配置阈值。
- 行业仓位超过配置阈值。
- 总仓位超过配置阈值。
- 当前价格超过最大可接受买入价。
- 标的是 ST、退市风险、停牌或交易异常。
- 连续亏损冷静期仍在生效。
- 存在未处理重大公告、财务或监管风险。

### 4.2 强提醒

触发后允许继续，但必须记录用户确认理由。

包括：

- 大盘环境较弱。
- 股票近期波动显著放大。
- 估值处于历史高分位。
- 所属行业近期明显走弱。
- 财报披露窗口临近。
- 买入理由主要依赖事件催化，且事件不确定性较高。
- 当前价格接近止损价，盈亏比不足。

### 4.3 信息提示

触发后只提示，不拦截。

包括：

- 同行业已有持仓。
- 标的近期公告较多。
- 标的近期涨幅较大但未超过追高阈值。
- 策略近期胜率下降。
- 计划持仓周期与历史平均持仓周期偏离。

## 5. 核心计算规则

### 5.1 单笔最大亏损

```text
单笔最大亏损比例 = 计划仓位 * abs(计划买入价 - 止损价) / 计划买入价
```

示例：

```text
计划仓位：10%
计划买入价：20
止损价：18

单笔最大亏损 = 10% * (20 - 18) / 20 = 1%
```

如果该值超过 `investment-profile` 中的 `max_loss_per_trade_pct_of_total_assets`，则阻断交易。

### 5.2 最大可接受买入价

```text
最大可接受买入价 = 计划买入价 * (1 + 追高阈值)
```

第一版默认追高阈值为 3%。

如果当前价格高于最大可接受买入价，则阻断交易。

### 5.3 仓位校验

系统需要检查：

- 本次计划仓位是否超过单票上限。
- 本次买入后行业总仓位是否超过行业上限。
- 本次买入后总仓位是否超过总仓位上限。
- 弱势市场下是否超过弱势市场总仓位上限。

## 6. 人工确认文案

风控通过后，真实交易前必须显示确认文案：

```text
我确认本次交易符合计划，已阅读关键证据、反证和风险，并接受按止损价计算的最大亏损。
```

用户确认后，交易计划才能进入“已执行”或“等待手动执行”状态。

## 7. 输出结果格式

风控校验结果应输出：

```text
交易计划编号：
校验结论：通过 / 阻断 / 需确认
阻断项：
强提醒项：
信息提示项：
计算结果：
  单笔最大亏损：
  买入后单票仓位：
  买入后行业仓位：
  买入后总仓位：
需要用户确认的事项：
```

## 8. 与复盘联动

交易计划中的字段会直接进入复盘。

复盘时必须对比：

- 实际买入价是否超过计划。
- 实际仓位是否超过计划。
- 是否按止损执行。
- 卖出原因是否符合计划。
- 买入理由是否被证实或证伪。
- 亏损是策略内亏损还是执行错误。

## 9. MVP 实现建议

第一版可以先实现为：

1. YAML 或 JSON 交易计划文件。
2. 命令行风控校验。
3. 输出阻断、强提醒和提示项。
4. 人工确认字段。
5. 后续再接 Web 表单和持仓系统。

对应示例文件：

- `templates/trade-plan.example.yaml`
