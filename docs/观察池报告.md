# 观察池报告

本文档定义第一版候选股观察池报告。

观察池报告面向人工决策，只回答“哪些股票值得观察、为什么、有什么风险、下一步看什么”。它不输出买入建议，不替代交易计划和风控校验。

## 1. 输入数据

默认输入：

```text
data/processed/candidate_pool.csv
```

该文件由 `tools/merge_candidate_pool.py` 生成。报告工具仍兼容旧版 `trend_candidates.csv`，但默认推荐使用统一候选池。

## 2. 输出数据

默认输出：

```text
reports/watchlist.md
```

真实观察池报告默认不提交到 Git。

## 3. 报告内容

报告包括：

- 报告生成时间。
- 候选股数量。
- 每只候选股的主策略、策略来源、策略数量、综合排序分。
- 趋势分、价值质量分、交易日、报告期。
- 入选原因。
- 风险提示。
- 下一步观察项。
- 生成交易计划的命令提示。

## 4. 生成命令

```bash
python3 tools/generate_watchlist_report.py \
  --candidates data/processed/candidate_pool.csv \
  --output reports/watchlist.md
```

使用样例链路：

```bash
python3 tools/calc_trend_factors.py \
  --daily-bars samples/daily_bars.sample.csv \
  --output /tmp/trend_factors.csv \
  --metadata-output /tmp/trend_factors.json \
  --windows 2

python3 tools/screen_trend_strength.py \
  --factors /tmp/trend_factors.csv \
  --output /tmp/trend_candidates.csv \
  --metadata-output /tmp/trend_candidates.json

python3 tools/screen_value_quality.py \
  --financial-metrics samples/financial_metrics.sample.csv \
  --output /tmp/value_quality_candidates.csv \
  --metadata-output /tmp/value_quality_candidates.json

python3 tools/merge_candidate_pool.py \
  --trend-candidates /tmp/trend_candidates.csv \
  --value-quality-candidates /tmp/value_quality_candidates.csv \
  --output /tmp/candidate_pool.csv \
  --metadata-output /tmp/candidate_pool.json

python3 tools/generate_watchlist_report.py \
  --candidates /tmp/candidate_pool.csv \
  --output /tmp/watchlist.md
```

## 5. 决策边界

观察池报告只允许输出：

- 入选原因。
- 风险提示。
- 观察条件。
- 交易计划生成提示。

禁止输出：

- 买入建议。
- 收益承诺。
- 绕过止损或仓位规则的建议。
- 无交易计划直接交易的提示。
