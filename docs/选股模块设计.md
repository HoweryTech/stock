# 选股模块设计

本文档定义个人 A 股投资决策系统中的选股模块边界、当前实现状态和后续扩展顺序。

选股模块的目标不是直接给出买入指令，而是生成可解释、可复盘、可进入交易计划门禁的观察池。真实交易仍必须由用户补全买入价、止损价、仓位和失效条件，并通过风控校验。

## 1. 决策边界

选股模块只回答：

- 哪些股票符合当前投资体系下的观察条件。
- 入选依据来自哪些策略和证据。
- 还缺少哪些反证、风险、流动性、行业或组合上下文。
- 是否具备进入交易计划草稿的最低证据完整性。

选股模块不回答：

- 是否应该立即买入。
- 应该买入多少。
- 是否绕过止损、仓位或组合约束。
- 是否自动下单。

## 2. 当前已实现链路

当前第一版链路以本地标准化 CSV 为输入：

```text
stock_universe.csv
  -> tradable_universe.csv

daily_bars.csv
  -> trend_factors.csv
  -> trend_candidates.csv
  -> industry_strength_factors.csv

financial_metrics.csv + valuation_metrics.csv
  -> value_quality_candidates.csv

event_catalyst_events.csv
  -> event_catalyst_candidates.csv

trend_candidates.csv + value_quality_candidates.csv + event_catalyst_candidates.csv
  -> candidate_pool.csv
  -> candidate_pool_check
  -> reports/watchlist.md
```

对应工具：

- `tools/filter_universe.py`：按投资体系过滤可交易股票池。
- `tools/calc_trend_factors.py`：计算基础趋势因子。
- `tools/calc_industry_strength.py`：按行业聚合收益、成交额和上涨占比，生成行业强度因子。
- `tools/screen_trend_strength.py`：筛选趋势强度候选。
- `tools/screen_value_quality.py`：筛选价值质量候选。
- `tools/screen_event_catalyst.py`：筛选结构化事件催化候选。
- `tools/merge_candidate_pool.py`：合并多策略候选池。
- `tools/check_candidate_pool.py`：检查候选池证据门禁。
- `tools/generate_watchlist_report.py`：生成观察池报告。
- `tools/prepare_trade_plan_from_candidate.py`：从候选进入交易计划门禁。

## 3. 股票池分层

选股模块按以下层级推进：

1. 全市场股票池：基础股票信息和行业分类。
2. 可交易股票池：排除 ST、退市风险、停牌、异常交易、上市天数不足、成交额过低等标的。
3. 风险过滤股票池：排除重大未处理公告、财务、监管或诚信风险。
4. 策略候选池：趋势强度、价值质量、事件催化、行业强度等策略独立产出候选。
5. 今日重点观察池：多策略合并、风险扣分、组合约束和人工复核后的观察列表。

当前代码已覆盖第 1、2、4、5 层的一部分；第 3 层和行业/事件/组合适配仍需扩展。

## 4. 候选输出结构

统一候选池至少保留：

- `code`：股票代码。
- `strategies`：命中的策略来源。
- `primary_strategy`：主策略或 `multi_strategy`。
- `combined_score`：观察池排序分。
- `trend_score`：趋势强度分。
- `value_quality_score`：价值质量分。
- `event_score`：事件催化分。
- `event_date`：事件日期。
- `event_type`：事件类型。
- `name`：股票名称。
- `industry`：所属行业。
- `liquidity_score`：流动性评分。
- `liquidity_evidence`：流动性证据。
- `trade_date`：趋势证据交易日。
- `report_period`：财务证据报告期。
- `reasons`：带策略前缀的入选证据。
- `risks`：带策略前缀的风险和反证。

后续应补充：

- `industry_strength_score`：行业强度评分。
- `industry_strength_evidence`：行业强度证据。
- `data_quality_status`：数据质量状态。
- `portfolio_fit_status`：组合适配状态。

## 5. 指标体系

### 5.1 硬过滤指标

- ST 或退市风险。
- 停牌或异常交易状态。
- 上市天数。
- 近 20 日平均成交额。
- 涨跌停状态。
- 数据是否缺失或过旧。

硬过滤不应被策略分数抵消。

### 5.2 基本面质量指标

- ROE、ROA。
- 毛利率、净利率。
- 经营现金流净额。
- 经营现金流与净利润匹配度。
- 资产负债率和有息负债。
- 营收同比。
- 净利润同比和扣非净利润同比。
- 财报异常、监管问询和财务更正。

### 5.3 估值指标

- PE TTM、PB、PS。
- PE/PB 历史分位。
- 行业估值分位。
- 股息率。
- PEG 或盈利增速与估值匹配度。

估值指标用于判断安全边际，不单独构成买入理由。

### 5.4 技术趋势指标

- 近 5/20/60 日收益率。
- MA5/10/20/60 结构。
- 是否站上关键均线。
- 相对大盘强度。
- 相对行业强度。
- 突破、回踩和支撑位。
- 成交额变化。
- 波动率和最大回撤。

### 5.5 行业与市场环境指标

- 行业指数近 5/20/60 日收益。
- 行业相对大盘强度。
- 行业内上涨家数占比。
- 行业成交额变化。
- 市场宽度。
- 弱市/强市仓位规则。

行业强度应作为下一阶段优先补齐的因子。

### 5.6 资金与交易活跃度指标

- 主力资金净流入。
- 大单净额。
- 换手率。
- 融资融券变化。
- 北向资金持股变化。
- 龙虎榜和大宗交易。

资金指标适合作为确认或风险提示，不宜单独成为买入理由。

### 5.7 事件催化指标

- 业绩预告。
- 回购。
- 增持或减持。
- 解禁。
- 重大合同。
- 政策催化。
- 监管问询。
- 重组和并购。

事件候选必须同时输出正向催化、反证、风险和失效条件。

### 5.8 组合适配指标

- 当前单票仓位。
- 所属行业已有仓位。
- 买入后总仓位。
- 现金比例。
- 单笔最大亏损。
- 同风格或同主题暴露。
- 是否处于亏损冷静期。

组合适配不改变股票本身质量，但决定候选能否进入交易计划。

## 6. 质量门禁

候选进入交易计划前，至少需要满足：

- 有股票代码、策略来源、主策略和入选证据。
- 趋势候选有交易日和趋势分。
- 价值质量候选有报告期和价值质量分。
- 如传入可交易股票池，候选必须仍在可交易股票池中。
- 至少具备两个非风险证据维度，或降级为人工复核。
- 有显式风险提示，否则降级为人工复核。
- 有流动性证据，否则降级为人工复核。

`blocked` 不应进入交易计划草稿；`needs_review` 可以进入人工研究，但不能直接视为可交易。

## 7. 后续实施顺序

建议按以下顺序迭代：

1. 固化当前设计文档和候选池质量门禁。
2. 为候选池补股票名称、行业和流动性评分。
3. 增加行业强度因子和行业相对强弱排序。
4. 增加事件催化候选文件并接入统一候选池。
5. 重构综合评分为策略共振、风险扣分、流动性、行业强度、数据质量和市场环境共同决定。
6. 接入组合适配，输出观察、可准备计划、因组合约束暂缓。
7. 建立候选入池后的 5/10/20 日表现复盘和策略健康度反馈。
