# 持仓准实时监控

`tools/monitor_intraday_positions.py` 使用东方财富公开网页行情快照持续监控持仓。接口当前实测通常为秒级，但没有时效、稳定性或授权等级承诺，因此只能用于观察和提醒，不能用于自动下单。

## 监控内容

- 最新价、涨跌幅、最高价、最低价、成交量和成交额。
- 接口行情时间戳与本地时间的实际滞后。
- 按成本价计算的实时浮盈亏和账户仓位。
- 相对5日、20日均线的位置。
- 涨跌停附近、日内大跌、单票仓位超限和行情过期状态。
- 反T观察状态、100股试做比例、卖出观察区、最低价差和最高回补价。
- 单票降至10%仓位以内所需的最少减仓股数、剩余股数和降仓后仓位。
- 10%仅是默认单票上限；未通过 `--position-limit-verified` 明确标记为用户确认前，程序只展示测算，不生成减仓执行信号。可通过 `--max-position-pct` 设置正式上限。
- 正式运行默认读取 `config/investment-profile.yaml`。当前已确认单票12%预警、15%硬上限、行业30%、总仓位80%、最低现金20%、单笔最大亏损1%、账户最大回撤10%，做T最低净收益10元，所有订单必须人工确认。

无论监控结果如何，当前版本始终输出：

- `add_allowed: false`
- `t_trade_allowed: false`
- `auto_order: false`

解除限制需要补齐止损、失效条件和盘中执行规则。

反T候选只代表价格位于当日波动区间上部且具备最低价差空间。执行前必须接受“未到回补价时不追买，可能形成实际减仓”的结果。持仓200股时允许使用100股，但标记为“单次涉及半仓，高风险”；持仓只有100股时因无法保留底仓而阻断。

反T费用模型默认使用保守假设：佣金率0.03%、每笔最低佣金5元、卖出印花税0.05%、双边过户费0.001%，并要求扣费后预计净收益至少5元。券商佣金和最低收费属于账户个性参数，未用交割单核验前，界面会标记“保守假设，尚未核验”。可通过监控命令的 `--commission-rate`、`--minimum-commission`、`--stamp-duty-rate`、`--transfer-fee-rate` 和 `--minimum-net-profit` 调整，核验后传入 `--cost-model-verified`。

页面净收益是算法找到的首个“扣费后达到最低净收益门槛”的情景，不是收益预测。小额交易通常同时触发买卖两笔最低佣金，因此不同股票的总费用和最低达标净收益会比较接近。

反T执行结论还必须通过 `tools/backtest_reverse_t.py` 的5分钟历史回测。回测在信号K线收盘后确认，最早使用下一根K线开盘价卖出，后续最低价实际触及回补上限才计为完成；未回补单独统计。样本不足或历史结果未通过时，页面禁止把反T候选显示为可执行。历史资金流、滑点和盘口排队尚未覆盖，因此通过回测也只能进入模拟盘观察。

`tools/forecast_reverse_t.py` 使用同一股票历史5分钟相似形态预测未来30分钟可能到达的高点区间。特征包含短周期收益、日内振幅位置、MACD、BOLL、RSI、成交量比、ATR和交易时段；输出到达概率和到达后形成扣费回补价差的概率。预测只负责提前预警，不能绕过历史回测和人工确认门禁。

价格进入反T卖出观察区时，页面立即显示“反T价格提醒”并记录状态事件，同时展示费用模型计算的回补参考上限。该提醒不等待回补价出现。3%主力资金流阈值仅作为风险说明，不再阻止价格到位提醒；回测门禁只限制执行可信度，不隐藏价格事实。

预警暂以“到达区间概率不低于60%，且在已经到达的相似样本中可回补概率不低于60%”为阈值。该阈值只控制提醒，不是交易许可。历史门禁至少要求30次触发、表面成功率不低于65%，且95% Wilson成功率下限不低于50%；之后仍只能进入模拟盘。

多周期趋势由日线自动聚合：使用5日/20日均线、4周/12周均价和3月/6月均价。周线、月线历史不足，或周月线共同偏弱且没有计划降仓目标时，系统阻断反T。

主力资金流来自东方财富按成交单大小划分的累计净额，包括超大单、大单、中单和小单。价格上涨但主力净流出会标记冲高分歧；价格下跌但主力净流入会标记承接观察。该指标不能识别具体机构或操盘主体，也不能单独作为交易依据。

### 反T资金流转弱与动态回落确认

反T卖出观察不能只用“主力净流入占比是否低于3%”这一条硬阈值。若主力净流入从6%以上逐步回落，等它低于3%时，价格可能已经明显离开卖出观察区。因此界面把资金流分成“绝对转弱”和“相对转弱”两层：

- `main_net_inflow`：东方财富实时行情字段 `f62`，表示按成交单大小统计的主力净流入金额。
- `main_net_inflow_ratio_pct`：东方财富实时行情字段 `f184 / 100`，表示主力净流入占成交额比例。
- `< 3%`：视为资金流绝对转弱确认。
- `3%-6%`：不再机械阻断；只有最近样本形成相对回落时，才视为资金流正在转弱。
- `> 6%`：视为强拉升区，反T不抢卖，等待进一步确认。

实时监控每轮会把价格、高点、主力净额和主力净流入占比追加到 `data/metadata/intraday-flow-history.jsonl`。本地界面通过 `/api/flow-history` 读取最近样本。资金流相对回落使用最近最多6个有效样本计算：

```text
values = 最近最多6个 main_net_inflow_ratio_pct
latest = values[-1]
peak = max(values)
slope = values 对样本序号的线性回归斜率
down_steps = 相邻样本中后一个值小于前一个值的次数
relative_drop_pct = (peak - latest) / max(abs(peak), 1) * 100
```

当同时满足以下条件时，`3%-6%` 区间可视为“资金流相对转弱”：

- `latest < peak`
- `slope < -0.15`，即最近样本整体向下。
- `down_steps >= ceil((样本数 - 1) * 0.5)`，即至少一半相邻样本在下降。
- `relative_drop_pct >= 15`

这套规则的含义是：不要求资金流一定跌破3%，但必须证明它不是单个噪声点，而是相对阶段高点出现连续回落。样本少于3个时，只提示“资金流历史样本不足”，不使用相对回落放行。

价格高点回落也不能写死为0.01元或0.02元。不同股票价格、最小卖出观察区宽度和日内波动都不同，固定价差会导致低价股过严、高价股过松。界面使用动态百分比确认：

```text
pullback_pct = (日内高点 - 当前价) / 日内高点 * 100
zone_width_pct = (卖出观察区上沿 - 卖出观察区下沿) / 卖出观察区上沿 * 100
required_pullback_pct = max(0.08, zone_width_pct * 0.5)
```

当 `当前价 < 日内高点` 且 `pullback_pct >= required_pullback_pct` 时，认为价格已经从高位形成动态回落。这里的 `0.08%` 是最低噪声过滤线，用于避免刚好低一跳就误判为回落；真正的确认线会随卖出观察区宽度动态变化。

因此，仓位超限降仓或反T相关的看盘解释应遵循：

- 价格到达卖出观察区时先提醒，不等待资金流和回补价。
- 若资金流 `< 3%`，可认为资金流转弱已经确认。
- 若资金流在 `3%-6%`，必须结合最近样本的相对回落趋势。
- 若资金流 `> 6%`，仍按强拉升处理，不在快速上冲中抢卖。
- 价格回落使用动态百分比确认，不写死为固定分钱数。
- 卖出后未到回补上限不追买；若事先接受降仓，未回补可计入计划降仓。

### 已优化的动态阈值

类似“3%资金流”这类规则，应优先从单点硬阈值改成“阈值 + 相对趋势 + 数据质量”的组合判断。当前已经落地以下动态规则：

- **新股有限样本模式**：上市时间较短、日线少于20根但不少于5根时，不再直接归类为“数据不足，暂不决策”。系统标记为 `limited_history`，允许基于实时行情、分钟线参考、费用模型和人工确认做保守分析；趋势、回测和多周期结论降级为参考，不作为放大交易依据。
- **正T/反T价格观察区间**：后台不再直接使用 `target_low + 0.02` 或 `high - 0.02`。观察区宽度使用 `ceil(锚定价格 * 0.18% / 0.01) * 0.01`，并限制在1-6个最小报价单位内。低价股区间会更窄，高价股区间会自然放宽。
- **价格源一致性阈值**：数据质量检查仍以配置阈值为基础，但会按价格档位动态微调。低于5元的股票容忍度上调0.4个百分点，5-10元上调0.2个百分点，50元及以上下调0.2个百分点，并在每条比对结果里输出 `allowed_diff_pct`。
- **正T分时评分门槛**：基础确认线仍为65分，但会按技术档位和确认信号数量动态调整。技术背景强且确认信号充分时可小幅降至62分；技术不可用、只观察或确认信号不足时提高确认线，避免仅靠单一分时信号触发。
- **反T预测概率阈值**：基础预警线仍为60%，但 `forecast_reverse_t.py` 会按样本数和相似样本数调整。样本少时提高到63%-68%，样本丰富时可小幅降低到55%-59%，并在结果中输出 `probability_policy`。

仍建议后续继续扩展的项：

- **做T阻断价附近判断**：当前依赖配置里的止损距离阈值。后续可引入“距止损百分比 + 日内波动 + ATR”的组合，避免单日大波动时误判。
- **技术解锁分数**：风险分、趋势分、量能分目前主要是固定目标线。后续可增加连续改善斜率，例如风险分从低位连续修复时先进入“接近解锁”，而不是只等一次性越线。
- **动作矩阵弱规则数量**：当前弱规则数量主要作为阻断/提示。后续可按同类市场状态、近期胜率和单票做T闭环结果加权，而不是只数规则条数。

## 数据质量快照

`tools/build_data_quality_snapshot.py` 用于在看盘中建议前检查数据是否足够可信。它会逐只持仓检查三类输入：

- 准实时行情：是否存在最新监控快照，以及行情延迟是否超过阈值。
- 日线数据：是否有足够样本，最新交易日是否过旧。
- 分钟线缓存：是否有足够5分钟样本，最新分钟线是否过旧。
- 行情源一致性：东方财富现价会与本地分钟线最新收盘价比对；若日线日期与分钟线日期相同，也会比对日线最新收盘价。日线日期落后时只作为参考，不直接判冲突。
- 交易时段：区分盘前、集合竞价、连续竞价、午间休市、尾盘集合竞价、盘后和非交易日。

历史数据会落到本地缓存：5分钟线按股票保存到 `data/processed/minute-bars/{code}.json`，日线统一合并到 `data/processed/daily_bars.csv`。

运行：

```bash
python3 tools/build_data_quality_snapshot.py \
  --positions 'positions/POS-EASTMONEY-*.yaml' \
  --intraday-snapshot data/metadata/intraday-monitor.latest.json \
  --daily-bars data/processed/daily_bars.csv \
  --minute-cache-dir data/processed/minute-bars
```

默认阈值：

- 行情延迟不超过60秒。
- 日线不少于20条，最新交易日距当前不超过5天。
- 分钟线不少于120条，最新分钟线距当前不超过30小时。
- 东方财富现价与可比缓存价差不超过1%，可通过 `--max-consistency-diff-pct` 调整。

输出：

- `data/metadata/data-quality-snapshot.json`
- `reports/data-quality-snapshot.md`

状态含义：

- `usable`：行情、日线和分钟线都可用于盘中判断。
- `stale`：数据存在但过期，只能复盘或观察，不能据此执行实时动作。盘前、午间休市、盘后或非交易日出现行情延迟时，会在决策卡中标记为等待交易时段，而不是盘中行情异常。
- `insufficient`：样本不足，必须补数据或降低该股票的自动建议置信度。
- `missing`：关键输入缺失，不能生成有效盘中建议。

可信等级：

- `高可信`：行情、日线、分钟线均满足阈值，允许进入盘中人工确认。
- `中可信`：行情可用，但日线或分钟线存在过期项，只能观察或复盘。
- `低可信`：行情过期、关键输入缺失、样本不足或多源价格明显冲突，必须先补数据或确认数据源。

## 动态止损复核与退出风控计划

动态止损复核价由实时决策卡根据当前行情、持仓止损草案、近期低点缓冲、ATR/波动、技术评分等信息计算。它是“本轮复核参考价”，不是永久固定价。后续开盘后行情、日线、分钟线或技术指标变化时，重新生成决策卡会重新计算动态复核价。

动态复核价有两种落地状态：

- **确认为硬止损**：把当前复核价写入持仓文件 `risk.stop_loss_price`，并标记 `risk.stop_loss_confirmed: true`。确认后的这个价会成为硬风控触发线，后续现价触及或跌破时进入退出风控流程。
- **仅保留参考**：把当前复核价写入持仓文件，但标记 `risk.stop_loss_confirmed: false` 和 `reference_only`。该价格只用于风险观察，不会作为卖出触发线，也不会把列表结论升级为止损退出。

即使某次动态复核价被确认为硬止损，后续仍允许再次调整。调整方式是重新点击页面里的“确认为硬止损”或“仅保留参考”，系统会记录新的确认时间、来源、原因和历史记录。硬止损确认不是永久锁死，而是把“本轮认可的风险边界”写入当前持仓计划。

### 盘中盯盘提醒

实时决策卡会输出 `intraday_trigger_alerts`，监控页面会把它们放在事件区最前面。提醒分两类：

- **已触发**：价格已经进入可执行动作或三路径预案的触发区，例如跌破硬止损、进入反抽减仓区或价格动作表出现 `ready`。
- **盯盘中**：已经形成盘中预案，但价格还没有触发路径；页面只提示需要盯住哪些价位，不提前下单。

近硬止损预案会重点盯三条路径：

- 路径1：现价小于等于硬止损，提醒进入止损减仓/硬退出复核，不再等待正T或反T信号。
- 路径2：现价进入反抽减仓区，提醒复核风控减仓数量和技术修复情况。
- 路径3：现价重新站上修复价且分钟确认通过，提醒本轮退出风险可降级为观察复核。

盯盘提醒只是把“该看哪条执行路径”推到页面前台，不自动下单。真正成交仍需要在券商软件里手动执行，并在本系统写入成交后刷新建议。

当提醒进入“已触发”状态时，监控页面会自动切换当前股票的展示重点：

- 列表唯一结论优先显示触发路径，而不是原来的等待结论。
- 点击股票会自动跳到对应的执行计划或可操作步骤区。
- 详情顶部操作条会显示“已触发”或“风险降级”，并展示当前触发价。
- 退出风控执行计划里的“当前结论”和“现在怎么做”会切换为对应路径的操作说明。

这个切换只改变页面展示优先级，不修改持仓文件、不写成交、不自动下单。

如果触发提醒属于“已触发”状态，页面会自动调用 `/api/intraday-trigger-refresh` 刷新完整日内决策链。刷新会重新生成准实时快照、数据质量、组合检查、做T检查、多周期技术指标和实时决策卡，使触发后的详情从“预案提示”尽快变成新的可执行计划或新的阻断结论。同一组股票/路径只会自动刷新一次，避免价格持续停留在触发区时反复跑管线。页面顶部会显示最近一次自动刷新的开始/结束时间、触发股票、新决策卡时间，以及“刷新前/刷新后”的结论差异，方便确认系统是否已经处理过本次触发。

每次自动刷新还会追加到 `data/metadata/intraday-trigger-refresh.events.jsonl`，页面事件区会显示最近的“自动刷新历史”。事件会记录触发路径、二次确认窗口、确认价、刷新前动作快照、刷新后动作快照和结论差异。如果价格触发后又离开触发区，可以通过这条历史记录确认当时触发过什么路径、为什么触发、刷新后系统给出的计划有没有变化。

页面还会基于这些历史事件生成“触发复核队列”。队列只显示当前需要处理的触发后事项，并按优先级分成：

- **需要处理**：刷新后主动作或人工计划已经进入可执行/风控减仓类状态，需要打开执行计划复核。
- **需要复核**：刷新后有决策卡结果，但还不能直接交易，需要查看阻断原因或复核结论。
- **只观察**：例如路径3站稳后风险降级，只需要继续观察，不做交易。

同一股票同一路径只保留最新一条，避免历史触发把当前待办列表刷满。

队列项支持标记处理结果，写入 `data/metadata/intraday-trigger-review-status.jsonl`：

- **已查看**：保留在队列中，但显示为已查看，表示已经打开过详情仍需继续跟踪。
- **已处理**：从默认待办队列移除，表示已按系统计划完成处理或确认无需继续挂起。
- **暂不处理**：从默认待办队列移除，表示当前明确不执行该事项，但保留审计记录。

当存在“已触发”的盘中提醒时，页面顶部会显示红色强提醒横幅，并把浏览器标题改为“盘中触发”。点击横幅里的“查看触发计划”会直接打开对应股票的执行计划区。如果浏览器通知权限已经允许，系统还会发送桌面通知；未授权时可在横幅里点击“启用桌面通知”。

为减少价格刚碰线又离开造成的误操作，近硬止损三路径触发会先进入页面运行态二次确认窗口：

- 路径1下破硬止损：连续满足约30秒后才视为确认触发。
- 路径2反抽减仓区：连续满足约60秒后才视为确认触发。
- 路径3站稳降级：连续满足约60秒后才视为确认触发。

确认窗口内页面会显示“等待二次确认”和剩余秒数，不触发自动刷新、不推送桌面强提醒、不提示直接下单。若价格在窗口内离开触发区，本次触发自动失效；若窗口结束时条件仍成立，才恢复为“已触发”，并启动自动刷新完整日内决策链。

触发硬止损后，系统不默认全仓退出，而是进入退出风控计划，并在详情页固定展示三段：

- **当前结论**：说明本轮是止损减仓、反弹减仓还是硬退出，以及为什么。
- **现在怎么做**：给出可直接照做的卖出方向、价格区间、数量和写入系统步骤。
- **做完会怎样**：展示成交后剩余股数、预计实现盈亏、是否还能做T和下一步观察计划。

退出风控计划分三类：

- **止损减仓**：跌破硬止损且技术继续偏弱时，优先卖出约50%风险仓位，至少100股；卖出后按剩余仓位重新评估，不用补仓或做T替代止损。
- **反弹减仓**：跌破硬止损但 RSI/KDJ/BOLL 等出现一定反转迹象，且风险没有继续恶化时，不追杀；等待反弹到系统给出的压力区，先卖出约50%风险仓位。
- **硬退出**：仅剩100股，或跌破止损超过约3%且趋势、量能继续偏弱时，才给出清仓式风控退出。

共通限制：

- 触发退出风控后禁止补仓、禁止正T、禁止反T摊低成本。
- 风控卖出不是反T卖出腿；成交后不要为了闭环而自动买回。
- 卖出后如果价格反弹，先等待系统按剩余仓位重新计算，不追高买回。
- 如果后续重新站回硬止损价，并且量能和趋势修复，可以重新评估做T或加仓；否则继续按风险优先处理。

### 日内操作节奏冷静期

系统会读取持仓文件里的 `manual_trade_history`，按当天日期统计手工成交次数、买卖方向、风控卖出次数和做T开/平腿次数，并把结果写入实时决策卡的 `daily_trade_rhythm`。

规则：

- 当天已经记录 `risk_exit_reduce` 或 `risk_exit_full` 时，进入“风控卖出后冷静”。
- 冷静期内禁止补仓、禁止正T买入、禁止新增反T卖出腿，也禁止把风控卖出当成反T回补式追买。
- 冷静期不阻止继续处理退出风险；如果价格继续跌破硬止损或触发减仓路径，仍按退出风控计划处理。
- 当天成交次数偏多或做T开/平腿次数偏多时，系统提示降频；只允许完成已打开的做T闭环或处理风险，不新增同方向试单。

页面详情会在操作步骤后展示“日内节奏”区块，列出今天已经成交几笔、当前禁止动作和下一步动作。若冷静期与技术候选冲突，人工候选计划会显示为 blocked，不再给出可直接执行的正T/反T计划。

## 实时决策卡

`tools/build_realtime_decision_cards.py` 将已有运行产物合并成每只持仓的统一决策卡：

- 准实时监控快照：现价、涨跌幅、行情延迟、盘中信号、反T观察计划。
- 数据质量快照：行情延迟、日线样本、分钟线缓存是否足够支撑盘中判断。
- 组合持仓日检：止损价、距止损比例、仓位风险。
- 做T机会检查：日线做T环境、阻断原因、正T/反T观察形态。
- 多周期技术指标：日线、周线、月线的 MACD、BOLL、RSI、KDJ、ATR 和量比，用于判断技术面偏多、偏弱、过热和波动风险。
- 动作矩阵回测：弱规则数量、历史表现较弱状态。
- 反T回测和预测：历史门禁、未来30分钟预警。

运行：

```bash
python3 tools/build_realtime_decision_cards.py \
  --intraday-snapshot data/metadata/intraday-monitor.latest.json \
  --portfolio-check data/metadata/eastmoney-portfolio-check.after-threshold.json \
  --t-opportunities data/metadata/eastmoney-portfolio-t-opportunities.near-config.json \
  --action-backtests data/metadata/portfolio-action-matrix-backtests.after-plan.json \
  --reverse-t-backtest data/metadata/reverse-t-backtest.json \
  --reverse-t-forecast data/metadata/reverse-t-forecast.json \
  --data-quality data/metadata/data-quality-snapshot.json \
  --technical-indicators data/metadata/technical-indicators.json
```

输出：

- `data/metadata/realtime-decision-cards.json`
- `reports/realtime-decision-cards.md`

卡片状态按保守优先级排序：退出风险、盘中行情或缓存过期、非交易时段等待、数据不足、仓位复核、技术面偏弱导致的持有不补仓、正T观察、反T观察、只观察。盘前、午间休市、盘后或非交易日的旧行情会显示为“非交易时段，等待行情”，仍禁止盘中动作，但不再误报为盘中数据异常。即使进入做T观察候选，默认仍是 `execution_allowed: false`，必须先生成交易或退出计划并人工确认。

本地监控界面会读取 `/api/decision-cards`。持仓列表的状态、建议和筛选优先使用决策卡结论，并展示每只股票的数据质量和可信等级标签；个股详情会展示关键价格、数据质量、可信等级、数据源一致性、技术指标判断、阻断原因和证据链。
