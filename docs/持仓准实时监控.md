# 持仓准实时监控

`tools/monitor_intraday_positions.py` 使用东方财富公开网页行情快照持续监控持仓。接口当前实测通常为秒级，但没有时效、稳定性或授权等级承诺，因此只能用于观察和提醒，不能用于自动下单。

## 监控内容

- 最新价、涨跌幅、最高价、最低价、成交量和成交额。
- 接口行情时间戳与本地时间的实际滞后。
- 按成本价计算的实时浮盈亏和账户仓位。
- 相对5日、20日均线的位置。
- 涨跌停附近、日内大跌、单票仓位超限和行情过期状态。
- 反T观察状态、100股试做比例、卖出观察区、最低价差和最高回补价。
- 单票降至10%仓位以内所需的最少减仓股数、剩余股数和降仓后仓位。
- 10%仅是默认单票上限；未通过 `--position-limit-verified` 明确标记为用户确认前，程序只展示测算，不生成减仓执行信号。可通过 `--max-position-pct` 设置正式上限。
- 正式运行默认读取 `config/investment-profile.yaml`。当前已确认单票12%预警、15%硬上限、行业30%、总仓位80%、最低现金20%、单笔最大亏损1%、账户最大回撤10%，做T最低净收益10元，所有订单必须人工确认。

无论监控结果如何，当前版本始终输出：

- `add_allowed: false`
- `t_trade_allowed: false`
- `auto_order: false`

解除限制需要补齐止损、失效条件和盘中执行规则。

反T候选只代表价格位于当日波动区间上部且具备最低价差空间。执行前必须接受“未到回补价时不追买，可能形成实际减仓”的结果。持仓200股时允许使用100股，但标记为“单次涉及半仓，高风险”；持仓只有100股时因无法保留底仓而阻断。

反T费用模型默认使用保守假设：佣金率0.03%、每笔最低佣金5元、卖出印花税0.05%、双边过户费0.001%，并要求扣费后预计净收益至少5元。券商佣金和最低收费属于账户个性参数，未用交割单核验前，界面会标记“保守假设，尚未核验”。可通过监控命令的 `--commission-rate`、`--minimum-commission`、`--stamp-duty-rate`、`--transfer-fee-rate` 和 `--minimum-net-profit` 调整，核验后传入 `--cost-model-verified`。

页面净收益是算法找到的首个“扣费后达到最低净收益门槛”的情景，不是收益预测。小额交易通常同时触发买卖两笔最低佣金，因此不同股票的总费用和最低达标净收益会比较接近。

反T执行结论还必须通过 `tools/backtest_reverse_t.py` 的5分钟历史回测。回测在信号K线收盘后确认，最早使用下一根K线开盘价卖出，后续最低价实际触及回补上限才计为完成；未回补单独统计。样本不足或历史结果未通过时，页面禁止把反T候选显示为可执行。历史资金流、滑点和盘口排队尚未覆盖，因此通过回测也只能进入模拟盘观察。

`tools/forecast_reverse_t.py` 使用同一股票历史5分钟相似形态预测未来30分钟可能到达的高点区间。特征包含短周期收益、日内振幅位置、MACD、BOLL、RSI、成交量比、ATR和交易时段；输出到达概率和到达后形成扣费回补价差的概率。预测只负责提前预警，不能绕过历史回测和人工确认门禁。

价格进入反T卖出观察区时，页面立即显示“反T价格提醒”并记录状态事件，同时展示费用模型计算的回补参考上限。该提醒不等待回补价出现。3%主力资金流阈值仅作为风险说明，不再阻止价格到位提醒；回测门禁只限制执行可信度，不隐藏价格事实。

预警暂以“到达区间概率不低于60%，且在已经到达的相似样本中可回补概率不低于60%”为阈值。该阈值只控制提醒，不是交易许可。历史门禁至少要求30次触发、表面成功率不低于65%，且95% Wilson成功率下限不低于50%；之后仍只能进入模拟盘。

多周期趋势由日线自动聚合：使用5日/20日均线、4周/12周均价和3月/6月均价。周线、月线历史不足，或周月线共同偏弱且没有计划降仓目标时，系统阻断反T。

主力资金流来自东方财富按成交单大小划分的累计净额，包括超大单、大单、中单和小单。价格上涨但主力净流出会标记冲高分歧；价格下跌但主力净流入会标记承接观察；主力净流入占比达到3%时，反T等待资金流转弱后再触发。该指标不能识别具体机构或操盘主体，也不能单独作为交易依据。

## 启动

```bash
python3 tools/monitor_intraday_positions.py \
  --positions 'positions/POS-EASTMONEY-*.yaml' \
  --daily-bars data/processed/daily_bars.csv \
  --total-assets 25480 \
  --interval 30 \
  --archive-interval 300
```

单次检查：

```bash
python3 tools/monitor_intraday_positions.py \
  --positions 'positions/POS-EASTMONEY-*.yaml' \
  --total-assets 25480 \
  --once
```

## 停止

```bash
python3 tools/stop_intraday_monitor.py
```

## 输出

- `reports/intraday-monitor.latest.md`：始终覆盖为最新可读报告。
- `data/metadata/intraday-monitor.latest.json`：最新结构化快照。
- `data/metadata/intraday-monitor.events.jsonl`：仅在状态或信号变化时追加。
- `data/metadata/intraday-archive/`：默认每5分钟归档一份完整快照。
- `data/metadata/intraday-monitor.pid`：运行中进程编号。

若行情时间戳滞后超过60秒，监控状态变为 `data_stale` 并停止给出有效盘中判断。

## 实时决策卡

`tools/build_realtime_decision_cards.py` 将已有运行产物合并成每只持仓的统一决策卡：

- 准实时监控快照：现价、涨跌幅、行情延迟、盘中信号、反T观察计划。
- 组合持仓日检：止损价、距止损比例、仓位风险。
- 做T机会检查：日线做T环境、阻断原因、正T/反T观察形态。
- 动作矩阵回测：弱规则数量、历史表现较弱状态。
- 反T回测和预测：历史门禁、未来30分钟预警。

运行：

```bash
python3 tools/build_realtime_decision_cards.py \
  --intraday-snapshot data/metadata/intraday-monitor.latest.json \
  --portfolio-check data/metadata/eastmoney-portfolio-check.after-threshold.json \
  --t-opportunities data/metadata/eastmoney-portfolio-t-opportunities.near-config.json \
  --action-backtests data/metadata/portfolio-action-matrix-backtests.after-plan.json \
  --reverse-t-backtest data/metadata/reverse-t-backtest.json \
  --reverse-t-forecast data/metadata/reverse-t-forecast.json
```

输出：

- `data/metadata/realtime-decision-cards.json`
- `reports/realtime-decision-cards.md`

卡片状态按保守优先级排序：退出风险、行情过期、数据不足、仓位复核、正T观察、反T观察、持有不补仓、只观察。即使进入做T观察候选，默认仍是 `execution_allowed: false`，必须先生成交易或退出计划并人工确认。

## 可视化界面

监控运行后启动本地界面：

```bash
python3 tools/serve_monitor_dashboard.py --host 127.0.0.1 --port 8765
```

浏览器访问 `http://127.0.0.1:8765`。界面每5秒读取最新监控快照，支持持仓风险筛选、代码/名称搜索、个股详情和状态变化事件查看。
