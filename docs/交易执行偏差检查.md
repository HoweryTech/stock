# 交易执行偏差检查

本文档定义交易执行记录的偏差检查规则。

执行偏差检查不判断股票未来涨跌，只判断真实或模拟成交是否偏离已经通过门禁的交易计划。

## 1. 输入

默认读取：

```text
executions/*.yaml
```

执行记录必须包含：

- 门禁结论。
- 人工确认状态。
- 成交价格。
- 实际仓位。
- 风控快照。
- 交易计划快照。

## 2. 阻断项

- 门禁结论不是 `pass` 或 `needs_confirmation`。
- 门禁需要人工确认但执行记录未确认。
- 真实执行未人工确认。
- 执行价格高于最大可接受买入价。
- 执行记录标记为超过最大可接受买入价。
- 实际仓位高于计划仓位。
- 执行价格缺失或不大于 0。

阻断项表示该执行记录不能直接生成持仓，需要人工修正或标记为执行错误。

## 3. 提醒项

- 执行价高于计划买入价，产生正滑点。

## 4. 信息项

- 执行价低于计划买入价。
- 实际仓位低于计划仓位。

## 5. 命令

```bash
python3 tools/check_trade_execution.py \
  --execution executions/EXEC-示例.yaml
```

输出 JSON：

```bash
python3 tools/check_trade_execution.py \
  --execution executions/EXEC-示例.yaml \
  --json
```
