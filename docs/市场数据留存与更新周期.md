# 市场数据留存与更新周期

本文档定义选股模块高优先级数据的拉取、留存和刷新节奏。原则是：原始抓取结果必须保留，`data/processed` 只放当前可用快照，后续接入数据库时以 `data/raw/snapshots` 作为回灌来源。

## 留存路径

- 股票池：`data/raw/snapshots/stock_universe/YYYY-MM-DD/stock_universe.HHMMSS.csv`
- 日线行情：`data/raw/snapshots/daily_bars/YYYY-MM-DD/*.csv`
- 估值快照：`data/raw/snapshots/valuation_metrics/YYYY-MM-DD/valuation_metrics.HHMMSS.csv`

这些文件被 `.gitignore` 排除，不提交到 Git，但会保留在本地磁盘。

## 高优先级刷新周期

| 数据 | 周期 | 建议时间 | 用途 |
| --- | --- | --- | --- |
| 股票池、交易状态、ST/停牌、当日成交额 | 每个交易日收盘后 | 17:10 | 更新可交易范围、流动性过滤和板块筛选 |
| 可交易股票池 | 股票池刷新后 | 17:15 | 排除 ST、退市风险、停牌、上市天数不足和流动性不足 |
| 日线 OHLCV | 每个交易日收盘后 | 17:30 | 趋势、技术指标、行业强度、回测 |
| PE TTM、PB、市值、流通市值 | 每周五收盘后 | 18:20 | 估值风险、排序辅助、价值质量策略 |
| PS、PCF、股息率 | 每周五收盘后 | 18:20 | 当前接口预留字段，需接更完整财务/估值源 |
| 行业估值分位 | 每月首个交易日收盘后 | 19:20 | 行业间估值比较和高估过滤 |
| 财务质量指标 | 季度刷新，财报季每日补拉 | 19:00 | ROE、营收增速、净利增速、现金流、负债率 |

## 盘前同步

盘前任务用于同步昨日收盘后的样本数据并重建候选池，默认不刷新估值：

```bash
.venv/bin/python tools/run_preopen_market_sync.py \
  --daily-datalen 10 \
  --workers 8 \
  --metadata-output data/metadata/preopen_market_sync.json
```

首次建库或需要修复历史样本时，把 `--daily-datalen` 提高到 `500`。如果当天也要刷新低频估值，可追加 `--refresh-valuation`。

macOS 本机定时任务可用以下命令安装，安装后工作日 08:40 自动执行盘前同步：

```bash
.venv/bin/python tools/install_preopen_launch_agent.py \
  --hour 8 \
  --minute 40 \
  --daily-datalen 10 \
  --workers 8
```

运行日志写入 `data/metadata/preopen_market_sync.launchd.log` 和 `data/metadata/preopen_market_sync.launchd.err`。

## 当前实现状态

- `tools/fetch_eastmoney_stock_universe.py` 已支持股票池抓取和自动留档。
- `tools/fetch_daily_bars_sina.py` 已支持日线抓取和自动留档。
- `tools/fetch_eastmoney_valuation_metrics.py` 已支持真实估值快照抓取和自动留档。
- `tools/run_preopen_market_sync.py` 已支持盘前同步股票池、可交易池、最近日线和候选池。
- `tools/install_preopen_launch_agent.py` 已支持安装 macOS 工作日 08:40 盘前同步任务。
- `config/market-data-schedule.example.yaml` 记录了建议定时任务。

当前东方财富行情列表可稳定提供 PE、PB、市值、流通市值。PS、PCF、股息率、行业估值分位先保留字段，不用不可靠映射填数，后续接财务报表源或历史估值分布后再补齐。
