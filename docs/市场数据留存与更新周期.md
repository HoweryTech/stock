# 市场数据留存与更新周期

本文档定义选股模块高优先级数据的拉取、留存和刷新节奏。原则是：原始抓取结果必须保留，`data/processed` 只放当前可用快照，后续接入数据库时以 `data/raw/snapshots` 作为回灌来源。

## 留存路径

- 股票池：`data/raw/snapshots/stock_universe/YYYY-MM-DD/stock_universe.HHMMSS.csv`
- 日线行情：`data/raw/snapshots/daily_bars/YYYY-MM-DD/*.csv`
- 估值快照：`data/raw/snapshots/valuation_metrics/YYYY-MM-DD/valuation_metrics.HHMMSS.csv`
- 财务指标：`data/raw/snapshots/financial_metrics/YYYY-MM-DD/financial_metrics.HHMMSS.csv`
- 公告/事件：`data/raw/snapshots/event_catalyst_events/YYYY-MM-DD/event_catalyst_events.HHMMSS.csv`

这些文件被 `.gitignore` 排除，不提交到 Git，但会保留在本地磁盘。

## 全量建库与增量更新

高优先级市场数据按“首次全量建库，后续增量更新”的方式维护。不要每天重复全量抓取高成本数据；只有在字段映射变更、数据源修复、历史窗口不足或本地快照损坏时才重新全量抓取。

| 数据 | 首次全量 | 后续增量 | 去重键 | 说明 |
| --- | --- | --- | --- | --- |
| 股票池、交易状态、ST/停牌、板块 | 每次全量快照 | 每个交易日全量快照 | `code + snapshot_date` | 数据量小，且状态变化重要，保留每日全量快照更利于复盘。 |
| 日线 OHLCV | 3-5 年，至少覆盖月线 MACD 所需窗口 | 每个交易日补最近 1-10 个交易日 | `code + trade_date` | 当前月线 MACD 至少需要 35 根月 K；首次建库建议拉 5 年。 |
| 估值快照 | 当前全市场快照 | 每周五或手动刷新追加快照 | `code + trade_date` | PE TTM、PB、市值、流通市值低频刷新即可；行情剧烈波动时可补跑。 |
| 财务指标 | 近 3-5 年报告期 | 财报季每日补最近报告期，平时季度刷新 | `code + report_period` | 支撑 ROE、ROA、现金流、负债率、营收/净利增长等价值质量策略。 |
| 公告/事件 | 近 6-12 个月 | 每日抓最近 1-7 天并按公告 ID 去重 | `announcement_id` | 原始公告先留存，再结构化为事件类型、影响评分、反证和风险披露。 |
| 行业估值分位 | 依赖估值历史窗口 | 每月首个交易日重算 | `industry + metric + as_of_date` | 由留存估值快照派生，数据源修复后可重算。 |

`data/processed` 中的文件是当前可用宽表，可以被覆盖；`data/raw/snapshots` 中的文件是审计和未来入库来源，不能因日常增量更新删除。

## 未来入库键设计

后续接入数据库时，优先按以下唯一键做 upsert：

| 表/数据 | 唯一键 | 主要用途 |
| --- | --- | --- |
| `stock_universe_snapshots` | `code + snapshot_date` | 复盘交易状态、板块、ST/停牌和流动性过滤。 |
| `daily_bars` | `code + trade_date` | 行情、技术指标、行业强度和回测。 |
| `valuation_metrics` | `code + trade_date` | PE/PB/市值、估值快照和估值分位计算。 |
| `financial_metrics` | `code + report_period` | 价值质量策略、多期财务对比和财务风险过滤。 |
| `announcements` | `announcement_id` | 公告去重、事件识别和人工复核状态。 |
| `event_catalyst_events` | `event_id` 或 `announcement_id + event_type` | 事件催化候选和风险公告过滤。 |
| `candidate_pool_snapshots` | `trade_date + code + strategy_version` | 每日候选留痕、入池后表现跟踪和策略复盘。 |

入库前仍保留 CSV/JSON 快照层，避免数据库迁移或表结构变化导致无法回放原始抓取结果。

## 高优先级刷新周期

| 数据 | 周期 | 建议时间 | 用途 |
| --- | --- | --- | --- |
| 股票池、交易状态、ST/停牌、当日成交额 | 每个交易日收盘后 | 17:10 | 更新可交易范围、流动性过滤和板块筛选 |
| 可交易股票池 | 股票池刷新后 | 17:15 | 排除 ST、退市风险、停牌、上市天数不足和流动性不足 |
| 日线 OHLCV | 每个交易日收盘后 | 17:30 | 趋势、技术指标、行业强度、回测 |
| PE TTM、PB、市值、流通市值 | 每周五收盘后 | 18:20 | 估值风险、排序辅助、价值质量策略 |
| PS、PCF、股息率 | 每周五收盘后 | 18:20 | 当前接口预留字段，需接更完整财务/估值源 |
| 行业估值分位 | 每月首个交易日收盘后 | 19:20 | 行业间估值比较和高估过滤 |
| 财务质量指标 | 季度刷新，财报季每日补拉 | 19:00 | ROE、营收增速、净利增速、现金流、负债率 |
| 公告/事件催化 | 每个交易日收盘后 | 18:40 | 回购、增持、业绩预告、重大订单、监管风险和其它事件识别 |

## 盘前同步

盘前任务用于同步昨日收盘后的样本数据并重建候选池，默认不刷新估值：

```bash
.venv/bin/python tools/run_preopen_market_sync.py \
  --daily-datalen 10 \
  --workers 8 \
  --metadata-output data/metadata/preopen_market_sync.json
```

首次建库或需要修复历史样本时，把 `--daily-datalen` 提高到 `500`。如果当天也要刷新低频估值，可追加 `--refresh-valuation`。

macOS 本机定时任务可用以下命令安装，安装后工作日 08:40 自动执行盘前同步：

```bash
.venv/bin/python tools/install_preopen_launch_agent.py \
  --hour 8 \
  --minute 40 \
  --daily-datalen 10 \
  --workers 8
```

运行日志写入 `data/metadata/preopen_market_sync.launchd.log` 和 `data/metadata/preopen_market_sync.launchd.err`。

## 当前实现状态

- `tools/fetch_eastmoney_stock_universe.py` 已支持股票池抓取和自动留档。
- `tools/fetch_daily_bars_sina.py` 已支持日线抓取和自动留档。
- `tools/fetch_eastmoney_valuation_metrics.py` 已支持真实估值快照抓取和自动留档。
- `tools/fetch_eastmoney_financial_metrics.py` 已支持真实财务指标抓取、增量合并和自动留档。
- `tools/run_preopen_market_sync.py` 已支持盘前同步股票池、可交易池、最近日线和候选池。
- `tools/install_preopen_launch_agent.py` 已支持安装 macOS 工作日 08:40 盘前同步任务。
- `config/market-data-schedule.example.yaml` 记录了建议定时任务。

当前东方财富行情列表可稳定提供 PE、PB、市值、流通市值。PS、PCF、股息率、行业估值分位先保留字段，不用不可靠映射填数，后续接财务报表源或历史估值分布后再补齐。

财务质量当前已有全市场批量抓取入口。下一阶段应先用 `--report-count 20` 做一次历史建库，再用 `--report-count 1` 做日常增量。公告/事件当前已有标准化输入、筛选逻辑和少量单股抓取能力，但还缺少全市场公告抓取、去重和事件分类；事件催化在接入时必须先做公告去重、事件分类和人工复核口径，避免只凭公告标题生成低质量推荐。
