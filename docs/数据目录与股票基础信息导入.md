# 数据目录与股票基础信息导入

本文档定义第一阶段本地数据目录规范，以及股票基础信息 CSV 的导入规则。

当前阶段先使用本地 CSV 作为数据入口，不直接绑定 AKShare、Tushare 或其他数据源。后续接入外部数据源时，也应先转换为本文档定义的标准结构，再进入选股和风控模块。

## 1. 数据目录

```text
data/
  raw/          原始导入文件，不直接给策略使用
  processed/    清洗后的标准数据，供股票池、选股和风控读取
  metadata/     导入元数据、数据质量报告、来源记录
```

真实数据文件默认不提交到 Git，只保留目录占位文件。

## 2. 股票基础信息 CSV

第一版股票基础信息文件至少包含以下字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| code | 是 | string | 股票代码，保留前导 0 |
| name | 是 | string | 股票名称 |
| exchange | 是 | string | 交易所，例如 SSE、SZSE、BSE |
| industry | 是 | string | 所属行业 |
| is_st | 是 | bool | 是否 ST |
| is_suspended | 是 | bool | 是否停牌 |
| has_delisting_risk | 是 | bool | 是否存在退市风险 |
| abnormal_trading_status | 是 | bool | 是否交易状态异常 |
| listing_date | 否 | date | 上市日期，格式 YYYY-MM-DD |
| avg_daily_turnover_cny | 否 | number | 近 20 日平均成交额，单位元 |
| data_source | 否 | string | 数据来源 |
| updated_at | 否 | date | 数据更新时间，格式 YYYY-MM-DD |

布尔字段支持：

```text
true / false
1 / 0
yes / no
y / n
是 / 否
```

## 3. 导入命令

```bash
python3 tools/import_stock_universe.py \
  --input samples/stock_universe.sample.csv \
  --output data/processed/stock_universe.csv \
  --metadata-output data/metadata/stock_universe.import.json
```

导入工具会执行：

- 必填字段检查。
- 股票代码格式校验。
- 布尔字段标准化。
- 数值字段标准化。
- 重复股票代码检查。
- 标准 CSV 输出。
- 导入元数据输出。

## 4. 输出标准

标准输出 CSV 字段顺序固定为：

```text
code,name,exchange,industry,is_st,is_suspended,has_delisting_risk,abnormal_trading_status,listing_date,avg_daily_turnover_cny,data_source,updated_at
```

后续基础风险过滤股票池会读取 `data/processed/stock_universe.csv`。
