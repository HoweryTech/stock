# 策略配置版本快照

本文档定义如何把当前投资体系配置固化为一个可复盘版本。

配置版本快照不修改配置，也不产生买卖建议。它只记录当前配置文件、配置哈希、关键风控边界、策略摘要，以及配置变更流水线、应用审计和回归检查的来源元数据。

## 1. 适用时机

在以下场景生成快照：

- 初始建立投资体系配置后。
- 配置变更流水线通过后。
- 手工调整配置并完成复核后。
- 需要把交易计划、执行记录、复盘或回测结果绑定到某一版投资体系前。

## 2. 默认输入

```text
config/investment-profile.example.yaml
data/metadata/strategy-config-change-pipeline.json
data/metadata/strategy-config-patch.apply.json
data/metadata/strategy-config-regression.json
```

后三个元数据文件缺失时仍可生成快照，报告会显式标记为“缺失”。

## 3. 默认输出

```text
reports/strategy-config-snapshot.md
data/metadata/strategy-config-snapshot.json
```

其中 JSON 结果可被后续日报、交易计划生成和回测归因读取；Markdown 结果用于人工审阅。

## 4. 命令

```bash
python3 tools/create_strategy_config_snapshot.py \
  --profile config/investment-profile.example.yaml \
  --pipeline data/metadata/strategy-config-change-pipeline.json \
  --audit data/metadata/strategy-config-patch.apply.json \
  --regression data/metadata/strategy-config-regression.json \
  --output reports/strategy-config-snapshot.md \
  --json-output data/metadata/strategy-config-snapshot.json
```

需要指定人工版本号时：

```bash
python3 tools/create_strategy_config_snapshot.py \
  --version-id CONFIG-VERSION-20260708-MANUAL
```

## 5. 快照字段

核心字段：

- `version_id`：配置版本号，默认按生成时间生成。
- `profile_hash`：对规范化 YAML 配置计算的 SHA256 哈希。
- `profile`：投资体系名称、市场范围、决策模式和主要目标。
- `risk`：关键风控边界。
- `strategies`：每个策略的启用状态、周期、证据数量和筛选字段。
- `source`：配置变更流水线、应用审计和回归检查的摘要。

## 6. 使用原则

- 如果 `profile_hash` 不同，就视为不同配置版本。
- 快照只能证明“当时使用了哪一版规则”，不能证明该规则一定有效。
- 交易计划、执行记录、复盘和回测应尽量记录对应 `version_id` 或 `profile_hash`，避免事后用新规则解释旧交易。
- 若配置回归结论为 `blocked`，不要把该快照作为新交易依据。
