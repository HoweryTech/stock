# 执行闭环总检查

本文档定义如何一次性检查买入执行、卖出执行和交易复盘记录的闭环状态。

该工具不产生买卖建议，只复用已有检查器汇总事实：哪些执行记录可以进入下一环节，哪些卖出执行不能生成复盘，哪些复盘记录仍缺少必要归因。

## 1. 输入

默认读取：

```text
executions/*.yaml
positions/*.yaml
exit-executions/*.yaml
reviews/*.yaml
```

## 2. 检查范围

- 买入执行记录：复用 `tools/check_trade_execution.py`。
- 卖出执行记录：复用 `tools/check_exit_execution.py`。
- 交易复盘记录：复用 `tools/check_trade_review_quality.py`。
- 下游记录完整性：通过检查的买入执行应生成持仓记录；通过检查的卖出执行应生成复盘记录。
- 孤儿记录：持仓必须能找到来源买入执行；复盘必须能找到来源卖出执行。

## 3. 结论

- `pass`：没有阻断记录，也没有需复核记录。
- `needs_review`：没有阻断记录，但存在需复核记录、缺失下游记录或孤儿记录。
- `blocked`：存在至少一条阻断记录，必须先修正后再推进下一环节。

报告会按 `blocked`、`needs_review`、`pass` 的优先级展示记录。缺失下游记录和孤儿记录会附带 `fix` 提示，用于给出最小修复动作。

报告还会生成 `修复动作分组`：

- `补齐人工确认`：确认快照缺失或未确认。
- `生成持仓记录`：买入执行通过但缺少持仓。
- `生成复盘记录`：卖出执行通过但缺少复盘。
- `修正来源引用`：持仓或复盘找不到来源记录。
- `修正阻断或复核记录`：其他字段、价格、仓位、归因等检查问题。

报告结构示例见：[执行闭环报告示例](./执行闭环报告示例.md)。

## 4. 命令

```bash
python3 tools/check_execution_loop.py \
  --output reports/execution-loop-check.md \
  --json-output data/metadata/execution-loop-check.json
```

只查看 JSON：

```bash
python3 tools/check_execution_loop.py --json
```

命令退出码规则：

- `0`：总结论不是 `blocked`。
- `1`：总结论为 `blocked`。
- `2`：读取或检查过程异常。
