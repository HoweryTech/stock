# 每日操作摘要

本文档定义如何生成每日操作摘要。

每日操作摘要不产生买卖建议，只汇总已经生成的观察池、组合持仓检查、退出计划、卖出执行、复盘草稿、策略健康、配置版本健康、策略复核任务、配置版本复核任务、策略配置变更草稿、待应用配置补丁、配置补丁应用审计、配置回归结果、配置变更流水线元数据和策略配置版本快照，帮助确认当天最该处理的事项。

## 1. 输入

默认读取：

```text
data/metadata/watchlist_pipeline.json
data/metadata/portfolio_positions.check.json
data/metadata/strategy-health.json
data/metadata/strategy-review-tasks.json
data/metadata/strategy-config-changes.json
data/metadata/strategy-config-patch.json
data/metadata/strategy-config-patch.apply.json
data/metadata/strategy-config-regression.json
data/metadata/strategy-config-change-pipeline.json
data/metadata/strategy-config-snapshot.json
data/metadata/manual-confirmations.json
exit-plans/*.yaml
exit-executions/*.yaml
reviews/*.yaml
```

缺少某类输入时，工具仍会生成报告，并把对应事项列入优先动作。

## 2. 输出

默认输出：

```text
reports/daily-summary.md
```

真实日报默认不提交到 Git。

## 3. 命令

```bash
python3 tools/generate_daily_summary.py \
  --output reports/daily-summary.md \
  --json-output data/metadata/daily-summary.json
```

如果只想看命令行 JSON：

```bash
python3 tools/generate_daily_summary.py --json
```

## 4. 优先动作

工具会根据当前材料生成操作提醒：

- 缺少观察池元数据：刷新观察池流水线。
- 缺少组合检查：执行组合持仓日检。
- 组合或持仓结论为 `needs_action`：优先处理风险。
- 存在紧急退出计划：先处理退出。
- 存在复盘草稿：补全复盘。
- 存在需复核配置版本：先确认问题来自配置规则、市场阶段还是具体策略。
- 存在未完成策略复核任务：先处理复核任务，再扩大对应策略风险暴露。
- 存在未完成配置版本复核任务：处理配置版本任务，再生成或审批配置变更草稿。
- 存在暂缓策略复核任务：复查暂缓原因是否仍成立。
- 存在待审批策略配置变更草稿：先审批或驳回，再修改投资体系配置。
- 存在待审批配置版本变更草稿：先审批或驳回，再进入配置补丁流程。
- 策略配置变更草稿已处理：显示已审批和已驳回数量，方便确认变更是否仍悬空。
- 存在待应用配置补丁：人工复核旧值、新值和来源证据，再决定是否修改配置。
- 存在配置补丁应用审计：显示应用人、应用时间、备份文件和已应用操作数。
- 配置回归检查阻断：先回滚或修复配置，不继续生成新计划。
- 配置回归检查需复核：先补齐审计或辅助信息，再视为闭环完成。
- 配置变更流水线阻断：先修正流水线中失败的阶段。
- 存在策略配置版本快照：显示当前配置版本、配置哈希和快照回归结论，便于后续交易计划、执行记录、复盘和回测引用同一版规则。
- 没有阻断项：保持观察，不因空闲而交易。

## 5. 必须人工确认事项

日报会在优先动作后单独输出 `今日必须人工确认事项`。该清单不是一般提醒，而是需要人工明确确认、审批、驳回或复核后才能继续推进的事项。

如果存在 `data/metadata/manual-confirmations.json`，日报会按稳定 `confirmation_id` 匹配确认记录，并显示：

- `待确认`：还没有确认记录。
- `已确认`：已有确认人和确认时间。
- `已驳回`：已有驳回人和驳回原因。

会进入该清单的事项包括：

- 紧急退出计划。
- 阻断级复盘修正结论。
- 暂停新开仓策略的执行边界。
- 待审批策略配置变更草稿。
- 待审批配置版本变更草稿。
- 待应用配置补丁。
- 配置回归或配置变更流水线阻断后的回滚、修复方案。

如果没有必须人工确认事项，日报会明确输出 `今日无必须人工确认事项。`
