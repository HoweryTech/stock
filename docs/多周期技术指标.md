# 多周期技术指标

本文档定义持仓技术指标计算层。它读取已经标准化的本地 OHLCV 日线数据，先聚合出周线和月线，再计算技术指标；行情源通常提供原始开高低收、成交量、成交额等数据，本系统不依赖外部接口直接给出 MACD、BOLL、RSI 等决策指标。

## 1. 数据输入

- `data/processed/daily_bars.csv`：标准化日线行情，至少需要 `trade_date`、`code`、`open`、`high`、`low`、`close`、`volume`。
- `positions/*.yaml`：可选。传入持仓文件后，只计算当前持仓股票；不传则计算日线文件中的全部股票。

## 2. 周期

- 日线：直接使用标准化日线。
- 周线：按 ISO 周聚合，开盘价取周内第一根，最高价取周内最高，最低价取周内最低，收盘价取周内最后一根，成交量和成交额求和。
- 月线：按自然月聚合，OHLCV 规则同周线。

## 3. 指标

第一版固定计算这些指标：

- `MACD(12,26,9)`：观察中期动能和柱体变化。
- `BOLL(20,2)`：观察价格在布林带中的位置、带宽和波动收敛/扩张。
- `RSI(6,14)`：短周期和常用周期的超买超卖参考。
- `KDJ(9,3,3)`：短线位置和钝化风险参考。
- `ATR(14)`：波动幅度，用于止损距离、做T空间和仓位风险评估。
- `volume_ratio_20`：最新成交量相对 20 根均量的量比，配合价格变化识别放量、缩量和量价背离。

已有的趋势强度基础因子仍保留，用于收益率、均线、成交额均值等候选池筛选；本工具提供更完整的持仓技术证据层。

MACD 输出包含最新 DIF、DEA、柱体、前一根 DIF/DEA/柱体、柱体变化和 `cross_status`。`cross_status` 用于识别 `golden_cross`、`dead_cross`、`bullish`、`bearish` 和 `turning_weak`，候选池会据此判断是否存在日线死叉或转弱趋势。

## 4. 运行方式

```bash
python3 tools/calc_technical_indicators.py \
  --daily-bars data/processed/daily_bars.csv \
  --positions 'positions/POS-EASTMONEY-*.yaml' \
  --output data/metadata/technical-indicators.json \
  --markdown-output reports/technical-indicators.md
```

输出：

- `data/metadata/technical-indicators.json`：结构化指标，供后续实时决策卡、日报和回测读取。
- `reports/technical-indicators.md`：人工可读摘要表。

## 5. 决策边界

本工具只计算技术指标，不直接生成买卖动作。最终动作仍需要结合实时行情可信度、止损/止盈价、持仓成本、做T阻断、仓位风险、事件和人工确认。

观察池流水线会读取 `data/metadata/technical-indicators.json` 并生成候选字段：

- `technical_health_score`：多周期技术健康分。
- `technical_health_status`：`strong`、`watch`、`weak`、`blocked` 或 `insufficient`。
- `technical_health_evidence`：技术证据摘要。
- `technical_risk_flags`：技术风险标记。

技术健康度是门禁/扣分型因子。日线转弱、MACD 死叉、BOLL 弱区、RSI/KDJ 偏弱、量比不足、ATR 过高会降低候选优先级；日线和周线同时转弱时会阻断进入买入计划。
