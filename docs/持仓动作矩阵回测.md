# 持仓动作矩阵回测

本文档定义如何用历史日线验证持仓趋势状态和价格动作矩阵。

该回测不模拟真实下单，不验证盘口成交、滑点、排队或盘中触发顺序。它只回答：

- 某个交易日以前的数据会把持仓归为哪种趋势状态。
- 当天动作矩阵会产生哪些关键触发规则。
- 触发后 1、3、5、10、20 个交易日的收益、最大上行和最大回撤如何。

## 命令

```bash
python3 tools/backtest_holding_action_matrix.py \
  --position positions/POS-示例.yaml \
  --daily-bars data/processed/daily_bars.csv \
  --output data/metadata/holding-action-matrix-backtest.json \
  --markdown-output reports/holding-action-matrix-backtest.md
```

指定观察窗口：

```bash
python3 tools/backtest_holding_action_matrix.py \
  --position positions/POS-示例.yaml \
  --horizons 1,3,5,10,20
```

## 输出

- `summary_by_trend_state`：按趋势状态统计后续收益和回撤。
- `summary_by_rule_trigger`：按动作矩阵触发规则统计后续收益和回撤。
- `recent_events`：最近触发事件明细。

## 边界

- 动作矩阵是风险和观察规则，不是自动买卖策略。
- 缺少止损价、失效条件或原始买入逻辑时，回测会如实反映规则被阻断，但不能替代人工补全计划。
- 回测结果只能用于评估规则质量，不代表未来表现。
