# 退出计划检查

本文档定义卖出执行前的退出计划检查。

退出计划检查不判断卖出后股价会涨还是跌，只判断这次卖出是否有计划、有证据、有仓位边界，并且能在事后复盘。

## 1. 输入

默认读取：

```text
exit-plans/*.yaml
```

退出计划必须包含：

- 退出计划编号和来源持仓编号。
- 退出类型和紧急度。
- 股票信息。
- 当前价格、当前仓位、计划退出价、计划退出仓位。
- 退出理由、退出证据、继续持有风险。
- 是否符合原交易计划。

## 2. 阻断项

- 缺少必要字段。
- 退出类型或紧急度不在允许范围内。
- 日检触发止损，但退出类型不是 `stop_loss`。
- 止损退出没有标记 `must_exit=true`。
- 计划退出价或最低可接受退出价不大于 0。
- 最低可接受退出价高于计划退出价。
- 计划退出仓位不大于 0。
- 计划退出仓位高于当前持仓。

阻断项表示该退出计划不能直接进入卖出执行。

## 3. 提醒项

- 止损退出未标记为 `immediate`。
- 标记为日检触发，但缺少日检结论。
- 退出计划不符合原交易计划。
- 计划退出价偏离当前价超过 5%。
- 退出类型为止盈，但当前收益率为负。

提醒项不一定禁止卖出，但需要人工确认和复盘标记。

## 4. 命令

```bash
python3 tools/check_exit_plan.py \
  --exit-plan exit-plans/EXIT-示例.yaml
```

输出 JSON：

```bash
python3 tools/check_exit_plan.py \
  --exit-plan exit-plans/EXIT-示例.yaml \
  --json
```
