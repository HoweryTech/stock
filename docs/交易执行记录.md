# 交易执行记录

本文档定义交易计划通过准入门禁和人工确认后的执行记录。

执行记录不是持仓记录。它只描述一次实际或模拟成交，用于保存计划门禁结论、人工确认、成交价格、滑点和执行备注。后续可以从执行记录生成持仓。

## 1. 输入

默认读取：

```text
plans/*.yaml
templates/trade-execution.example.yaml
config/investment-profile.example.yaml
```

可选读取：

```text
data/metadata/*.gate.json
data/metadata/review-cooldown.json
data/metadata/strategy-health.json
data/metadata/strategy-config-snapshot.json
```

如果不传 `--gate`，工具会现场执行交易计划准入门禁。
如果存在 `data/metadata/review-cooldown.json`，工具会读取复盘冷静期检查结论。
如果存在 `data/metadata/strategy-health.json`，工具会读取策略健康检查结论。
如果交易计划尚未记录 `strategy_config_snapshot`，工具会尝试读取 `data/metadata/strategy-config-snapshot.json` 并补入执行记录中的 `trade_plan_snapshot`。

## 2. 执行约束

允许生成执行记录的门禁结论：

- `pass`
- `needs_confirmation`

当门禁结论为 `needs_confirmation` 时，必须传入 `--user-confirmed`。

真实执行 `--mode real` 必须传入 `--user-confirmed`。

阻断结论不允许生成执行记录。

当冷静期结论为 `cooldown_required` 时，买入执行默认阻断。若确实需要例外交易，必须同时传入：

- `--allow-cooldown-exception`
- `--user-confirmed`
- `--cooldown-exception-reason`

卖出执行不受买入冷静期阻断。

当当前交易计划策略在策略健康检查中为 `pause_new_entries` 时，买入执行默认阻断。确需例外时，复用同一组例外参数，并在原因中说明为什么可以违反策略暂停纪律。

## 3. 命令

模拟执行：

```bash
python3 tools/new_trade_execution.py \
  --plan plans/TP-示例.yaml \
  --strategy-config-snapshot data/metadata/strategy-config-snapshot.json \
  --mode paper \
  --execution-date 2026-07-07 \
  --execution-price 200 \
  --shares 100 \
  --position-pct 5 \
  --user-confirmed \
  --note "按计划模拟成交。"
```

使用已保存门禁结果：

```bash
python3 tools/new_trade_execution.py \
  --plan plans/TP-示例.yaml \
  --gate data/metadata/TP-示例.gate.json \
  --mode paper \
  --execution-date 2026-07-07 \
  --execution-price 200 \
  --user-confirmed
```

冷静期例外买入：

```bash
python3 tools/new_trade_execution.py \
  --plan plans/TP-示例.yaml \
  --cooldown-check data/metadata/review-cooldown.json \
  --strategy-health data/metadata/strategy-health.json \
  --allow-cooldown-exception \
  --cooldown-exception-reason "例外原因：小仓位验证，不扩大总风险。" \
  --mode paper \
  --execution-date 2026-07-07 \
  --execution-price 200 \
  --user-confirmed
```

## 4. 输出

默认输出：

```text
executions/EXEC-YYYYMMDD-HHMMSS.yaml
```

真实执行记录默认不提交到 Git。
