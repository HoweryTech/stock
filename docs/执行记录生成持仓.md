# 执行记录生成持仓

本文档定义从交易执行记录生成持仓记录的规则。

持仓不应直接从交易计划生成。标准流程应为：

```text
交易计划
  -> 准入门禁
  -> 执行记录
  -> 持仓记录
```

这样可以在复盘时区分计划问题和执行问题。

## 1. 输入

默认读取：

```text
executions/*.yaml
templates/position.example.yaml
```

执行记录必须包含 `trade_plan_snapshot`。
生成持仓前会自动执行交易执行偏差检查。若执行检查结论为 `blocked`，默认不生成持仓。

## 2. 继承字段

持仓会从执行记录继承：

- 成交日期。
- 成交价格。
- 成交数量。
- 实际仓位。
- 止损价。
- 交易计划快照。
- 策略配置版本快照。
- 执行记录快照。
- 执行偏差检查快照。

持仓会把原交易计划里的 `strategy_config_snapshot` 提到顶层，便于按 `version_id` 或 `profile_hash` 检索；完整交易计划仍保存在 `trade_plan_snapshot`。

## 3. 命令

```bash
python3 tools/new_position_from_execution.py \
  --execution executions/EXEC-示例.yaml \
  --current-price 201 \
  --days-held 1 \
  --note "从执行记录生成持仓。"
```

如需处理历史修正或错误归档，可以显式允许阻断执行进入持仓：

```bash
python3 tools/new_position_from_execution.py \
  --execution executions/EXEC-示例.yaml \
  --allow-blocked-execution
```

默认输出：

```text
positions/POS-YYYYMMDD-HHMMSS.yaml
```

真实持仓记录默认不提交到 Git。
