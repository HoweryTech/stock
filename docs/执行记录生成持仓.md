# 执行记录生成持仓

本文档定义从交易执行记录生成持仓记录的规则。

持仓不应直接从交易计划生成。标准流程应为：

```text
交易计划
  -> 准入门禁
  -> 执行记录
  -> 持仓记录
```

这样可以在复盘时区分计划问题和执行问题。

## 1. 输入

默认读取：

```text
executions/*.yaml
templates/position.example.yaml
```

执行记录必须包含 `trade_plan_snapshot`。

## 2. 继承字段

持仓会从执行记录继承：

- 成交日期。
- 成交价格。
- 成交数量。
- 实际仓位。
- 止损价。
- 交易计划快照。
- 执行记录快照。

## 3. 命令

```bash
python3 tools/new_position_from_execution.py \
  --execution executions/EXEC-示例.yaml \
  --current-price 201 \
  --days-held 1 \
  --note "从执行记录生成持仓。"
```

默认输出：

```text
positions/POS-YYYYMMDD-HHMMSS.yaml
```

真实持仓记录默认不提交到 Git。
