# 执行闭环修复任务

本文档定义如何把执行闭环总检查中的 `fix_actions` 转成待办清单。

该工具不自动修改任何交易、持仓、卖出或复盘记录，只把可执行修复动作结构化，方便逐条处理。

## 1. 输入

默认读取：

```text
data/metadata/execution-loop-check.json
```

## 2. 输出

默认输出：

```text
reports/execution-fix-tasks.md
data/metadata/execution-fix-tasks.json
```

真实任务清单默认不提交到 Git。

## 3. 任务来源

任务来自执行闭环总检查的 `fix_actions`，例如：

- 补齐人工确认。
- 生成持仓记录。
- 生成复盘记录。
- 修正来源引用。
- 修正阻断或复核记录。

## 4. 命令

```bash
python3 tools/generate_execution_fix_tasks.py \
  --loop-check data/metadata/execution-loop-check.json \
  --output reports/execution-fix-tasks.md \
  --json-output data/metadata/execution-fix-tasks.json
```

命令退出码规则：

- `0`：没有待处理任务。
- `1`：存在待处理任务。
- `2`：读取或生成过程异常。
