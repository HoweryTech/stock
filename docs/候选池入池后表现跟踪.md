# 候选池入池后表现跟踪

本文档定义观察池候选入池后的前向表现跟踪规则。

该工具只评估候选进入观察池后的价格表现，不评价是否应该真实买入，也不替代交易计划和风控门禁。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
data/processed/daily_bars.csv
```

候选入池日期优先使用：

1. `trade_date`
2. `event_date`
3. `valuation_trade_date`

## 2. 输出

默认输出：

```text
data/metadata/candidate-performance.json
reports/candidate-performance.md
```

输出包含：

- 候选数量。
- 每个观察窗口的完成数量。
- 平均收益率。
- 胜率。
- 每只候选的入池日期、入池价和各窗口收益。

该 JSON 会被策略健康检查作为可选观察输入读取，并按策略汇总到 `candidate_observation`。这只用于复查选股入口质量，不代表真实交易收益。

## 3. 命令

```bash
python3 tools/track_candidate_performance.py \
  --candidates data/processed/candidate_pool.csv \
  --daily-bars data/processed/daily_bars.csv \
  --horizons 5,10,20 \
  --output data/metadata/candidate-performance.json \
  --markdown-output reports/candidate-performance.md
```
