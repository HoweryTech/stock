# 交易计划补全辅助

本文档定义交易计划草稿的命令行补全工具。

候选池生成的交易计划草稿会自动带入证据和风险，但不会替用户猜测止损条件、止盈条件、失效条件和复盘关注点。`tools/complete_trade_plan.py` 用于补齐这些人工决策字段，并重新计算衍生风控字段。

## 1. 输入

默认读取真实交易计划：

```text
plans/*.yaml
```

真实交易计划默认不提交到 Git。

## 2. 可补全字段

工具支持补充或更新：

- 买入理由。
- 关键证据。
- 反证和风险。
- 止损条件。
- 止盈条件。
- 失效条件。
- 观察项。
- 复盘关注点。
- 计划买入价、当前价、止损价。
- 计划仓位、当前单票仓位、行业仓位、总仓位。

价格和仓位更新后，工具会重新计算：

- 最大可接受买入价。
- 买入后单票仓位。
- 买入后行业仓位。
- 买入后总仓位。
- 单笔最大亏损。

## 3. 命令

```bash
python3 tools/complete_trade_plan.py \
  --plan plans/TP-示例.yaml \
  --stop-loss-condition "收盘价跌破止损价。" \
  --take-profit-condition "达到计划目标区后根据量价分批止盈。" \
  --invalidation-condition "趋势或基本面证据失效。" \
  --review-focus "候选池证据是否被市场验证。" \
  --mark-ready
```

默认会原地更新 `--plan` 文件。如需输出到新文件：

```bash
python3 tools/complete_trade_plan.py \
  --plan plans/TP-示例.yaml \
  --output /tmp/TP-示例.completed.yaml \
  --stop-loss-condition "收盘价跌破止损价。"
```

## 4. 准入门禁

补全后继续执行：

```bash
python3 tools/check_trade_plan_gate.py \
  --profile config/investment-profile.example.yaml \
  --plan plans/TP-示例.yaml
```
