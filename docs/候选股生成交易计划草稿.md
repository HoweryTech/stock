# 候选股生成交易计划草稿

本文档定义从观察池候选股生成交易计划草稿的规则。

该功能只负责把候选池证据带入交易计划模板，不自动生成买入结论。计划中的买入价、止损价、仓位、失效条件仍必须由人工填写或确认。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
templates/trade-plan.example.yaml
config/investment-profile.example.yaml
```

候选池由 `tools/merge_candidate_pool.py` 或 `tools/run_watchlist_pipeline.py` 生成。

## 2. 预填规则

工具会根据候选池行预填：

- 股票代码。
- 策略来源。
- 买入理由草稿。
- 关键证据。
- 反证和风险。
- 下一步观察项。

多策略候选默认使用候选池中的第一个策略作为交易计划 `strategy.source`。可以通过 `--strategy` 手工覆盖。

候选池没有显式风险时，工具会写入兜底风险提示，要求人工补充估值、公告、市场环境、止损和仓位风险。

## 3. 不自动填写的字段

以下字段不会从候选池里猜测：

- 股票名称。
- 交易所。
- 行业。
- 计划买入价。
- 当前价。
- 止损价。
- 计划仓位。
- 止盈条件。
- 失效条件。

这些字段必须人工补充。缺失时，后续风控校验会阻断或要求确认。

## 4. 命令

```bash
python3 tools/new_trade_plan_from_candidate.py \
  --candidates data/processed/candidate_pool.csv \
  --code 300750 \
  --name 宁德时代 \
  --exchange SZSE \
  --industry 电力设备 \
  --planned-buy-price 200 \
  --stop-loss-price 185 \
  --position-pct 5 \
  --stop-loss-condition "收盘价跌破止损价。" \
  --take-profit-condition "达到计划目标区后根据量价分批止盈。" \
  --invalidation-condition "趋势或基本面证据失效。"
```

真实交易计划默认输出到 `plans/`，并且默认不提交到 Git。

## 5. 决策边界

生成草稿后必须继续执行：

```bash
python3 tools/risk_check.py \
  --profile config/investment-profile.example.yaml \
  --plan plans/TP-示例.yaml
```

风控校验未通过前，不应执行交易。
