# 卖出执行记录

本文档定义退出计划通过检查后的卖出执行记录。

卖出执行记录用于回答：实际卖出价格是多少、卖出了多少仓位、是否低于最低可接受退出价、结果对组合收益的影响是多少。

## 1. 输入

默认读取：

```text
exit-plans/*.yaml
templates/exit-execution.example.yaml
data/metadata/manual-confirmations.json
```

工具会自动运行退出计划检查。检查结论为：

- `pass`：可以生成卖出执行记录。
- `needs_review`：必须存在 `confirmed` 状态的人工确认记录后才能生成。
- `blocked`：不能生成。

## 2. 风控约束

- 真实卖出必须传入 `--confirmation-id`，并且该 ID 在 `data/metadata/manual-confirmations.json` 中为 `confirmed`。
- `needs_review` 必须传入 `--confirmation-id`，并且该确认记录会写入 `confirmation_snapshot`。
- 仅传入 `--user-confirmed` 不满足真实卖出或 `needs_review` 的审计要求；它只保留为兼容字段。
- 成交价低于最低可接受退出价时，默认阻断；确实需要记录偏差时，传入 `--allow-below-min-price`。

## 3. 命令

```bash
python3 tools/new_exit_execution.py \
  --exit-plan exit-plans/EXIT-示例.yaml \
  --execution-date 2026-07-08 \
  --execution-price 9.1 \
  --shares 1000 \
  --output exit-executions/EXITEXEC-示例.yaml
```

真实卖出：

```bash
python3 tools/new_exit_execution.py \
  --exit-plan exit-plans/EXIT-示例.yaml \
  --mode real \
  --confirmation-id CONFIRM-EXIT-示例 \
  --execution-date 2026-07-08 \
  --execution-price 9.1
```

卖出执行记录会继承退出计划的 `strategy_config_snapshot`，并保留完整 `exit_plan_snapshot` 和 `confirmation_snapshot`。后续复盘可直接按顶层配置版本、退出计划和人工确认记录归因。
