# 策略配置变更记录

本文档定义如何从已解决的策略复核任务生成策略配置变更草稿。

该流程不自动修改 `config/investment-profile.example.yaml`。它只把复核任务中的人工结论、证据和建议变更项整理成可审计草稿，避免策略规则被临时、口头或无证据地修改。

## 1. 输入

默认读取：

```text
data/metadata/strategy-review-tasks.json
```

只有 `task_status: resolved` 且 `resolution` 非空的任务会生成变更草稿。

## 2. 输出

默认输出：

```text
reports/strategy-config-changes.md
data/metadata/strategy-config-changes.json
```

真实变更记录默认不提交到 Git。

## 3. 变更草稿内容

每条草稿包含：

- 来源策略复核任务 ID。
- 策略名称。
- 人工复核结论。
- 来源动作、统计数据和解决时间。
- 建议变更路径。
- 建议变更内容。
- 审批字段。
- 历史记录。

新生成的草稿默认：

- `status: draft`
- `approval.required: true`
- `approval.approved_by: ""`
- `approval.rejected_by: ""`
- `approval.rejected_reason: ""`
- `history: []`

## 4. 变更映射

工具会按复核任务动作生成初始建议：

- `strategy_cooldown_required`：复核是否暂停该策略新开仓，以及恢复条件。
- `loss_making_discipline_exception`：降低或取消纪律例外仓位上限。
- `low_win_rate`、`low_average_return`、`negative_portfolio_contribution`：提高筛选阈值、缩小适用场景或降低默认仓位。

未映射的动作会生成策略级占位变更，要求人工补齐具体字段。

## 5. 命令

```bash
python3 tools/generate_strategy_config_changes.py \
  --tasks data/metadata/strategy-review-tasks.json \
  --output reports/strategy-config-changes.md \
  --json-output data/metadata/strategy-config-changes.json
```

## 6. 审批或驳回

配置变更草稿支持三种动作：

- `approve`：审批通过，可填写生效日期。
- `reject`：驳回，必须填写原因。
- `reopen`：重新打开，回到待审批草稿。

审批命令：

```bash
python3 tools/update_strategy_config_change.py \
  --changes data/metadata/strategy-config-changes.json \
  --change-id CONFIG-CHANGE-STRATEGY-REVIEW-TREND-STRENGTH-NEEDS-REVIEW \
  --action approve \
  --actor lihongwei \
  --reason "复核通过。" \
  --effective-date 2026-07-09
```

驳回命令：

```bash
python3 tools/update_strategy_config_change.py \
  --changes data/metadata/strategy-config-changes.json \
  --change-id CONFIG-CHANGE-STRATEGY-REVIEW-TREND-STRENGTH-NEEDS-REVIEW \
  --action reject \
  --actor lihongwei \
  --reason "证据不足，暂不修改策略规则。"
```

审批或驳回只更新变更记录状态，不修改真实配置文件。
