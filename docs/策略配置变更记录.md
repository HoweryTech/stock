# 策略配置变更记录

本文档定义如何从已解决的策略复核任务生成策略配置变更草稿。

该流程不自动修改 `config/investment-profile.example.yaml`。它只把复核任务中的人工结论、证据和建议变更项整理成可审计草稿，避免策略规则被临时、口头或无证据地修改。

## 1. 输入

默认读取：

```text
data/metadata/strategy-review-tasks.json
```

只有 `task_status: resolved` 且 `resolution` 非空的任务会生成变更草稿。

## 2. 输出

默认输出：

```text
reports/strategy-config-changes.md
data/metadata/strategy-config-changes.json
```

真实变更记录默认不提交到 Git。

## 3. 变更草稿内容

每条草稿包含：

- 来源策略复核任务 ID。
- 策略名称。
- 人工复核结论。
- 来源动作、统计数据和解决时间。
- 建议变更路径。
- 建议变更内容。
- 审批字段。
- 历史记录。

新生成的草稿默认：

- `status: draft`
- `approval.required: true`
- `approval.approved_by: ""`
- `approval.rejected_by: ""`
- `approval.rejected_reason: ""`
- `history: []`

## 4. 变更映射

工具会按复核任务动作生成初始建议：

- `strategy_cooldown_required`：复核是否暂停该策略新开仓，以及恢复条件。
- `loss_making_discipline_exception`：降低或取消纪律例外仓位上限。
- `low_win_rate`、`low_average_return`、`negative_portfolio_contribution`：提高筛选阈值、缩小适用场景或降低默认仓位。

未映射的动作会生成策略级占位变更，要求人工补齐具体字段。

## 5. 命令

```bash
python3 tools/generate_strategy_config_changes.py \
  --tasks data/metadata/strategy-review-tasks.json \
  --output reports/strategy-config-changes.md \
  --json-output data/metadata/strategy-config-changes.json
```

## 6. 审批或驳回

配置变更草稿支持三种动作：

- `approve`：审批通过，可填写生效日期。
- `reject`：驳回，必须填写原因。
- `reopen`：重新打开，回到待审批草稿。

审批命令：

```bash
python3 tools/update_strategy_config_change.py \
  --changes data/metadata/strategy-config-changes.json \
  --change-id CONFIG-CHANGE-STRATEGY-REVIEW-TREND-STRENGTH-NEEDS-REVIEW \
  --action approve \
  --actor lihongwei \
  --reason "复核通过。" \
  --effective-date 2026-07-09
```

驳回命令：

```bash
python3 tools/update_strategy_config_change.py \
  --changes data/metadata/strategy-config-changes.json \
  --change-id CONFIG-CHANGE-STRATEGY-REVIEW-TREND-STRENGTH-NEEDS-REVIEW \
  --action reject \
  --actor lihongwei \
  --reason "证据不足，暂不修改策略规则。"
```

审批或驳回只更新变更记录状态，不修改真实配置文件。

## 7. 应用前校验

配置变更进入应用前，必须先执行校验：

```bash
python3 tools/check_strategy_config_changes.py \
  --profile config/investment-profile.example.yaml \
  --changes data/metadata/strategy-config-changes.json \
  --output data/metadata/strategy-config-changes.check.json
```

校验会检查：

- 变更是否已审批。
- 变更路径是否存在。
- 是否填写 `proposed_value`。
- `proposed_value` 类型是否与当前配置一致。
- 风险参数是否超出安全边界。

校验结论：

- `blocked`：不能应用。
- `needs_review`：需要补齐信息后再应用。
- `pass`：可以进入生成待应用补丁阶段。

## 8. 生成待应用补丁

校验结论为 `pass` 后，可以生成待应用配置补丁：

```bash
python3 tools/generate_strategy_config_patch.py \
  --profile config/investment-profile.example.yaml \
  --changes data/metadata/strategy-config-changes.json \
  --check data/metadata/strategy-config-changes.check.json \
  --output reports/strategy-config-patch.md \
  --json-output data/metadata/strategy-config-patch.json
```

补丁只列出 `replace` 操作、旧值、新值、来源任务和原因，不自动修改配置文件。

## 9. 应用配置补丁

人工复核待应用补丁后，可以应用到配置文件：

```bash
python3 tools/apply_strategy_config_patch.py \
  --profile config/investment-profile.example.yaml \
  --patch data/metadata/strategy-config-patch.json \
  --backup-dir data/backups \
  --audit-output data/metadata/strategy-config-patch.apply.json \
  --applied-by lihongwei
```

应用工具会执行以下保护：

- 要求补丁 `check_conclusion` 为 `pass`。
- 要求补丁 `apply_mode` 为 `manual_review_required`。
- 要求填写 `--applied-by`。
- 应用前确认当前配置值与补丁 `old_value` 一致。
- 修改前备份原配置。
- 写出应用审计记录。

## 10. 应用后回归检查

配置补丁应用后，执行回归检查：

```bash
python3 tools/check_strategy_config_regression.py \
  --profile config/investment-profile.example.yaml \
  --audit data/metadata/strategy-config-patch.apply.json \
  --output data/metadata/strategy-config-regression.json
```

回归检查会验证：

- 必需配置段仍存在。
- 风险参数仍在安全边界内。
- 首选策略都有对应配置。
- 策略 `enabled`、`required_evidence`、`screening` 等基础结构仍可解析。
- 应用审计记录存在且操作数量一致。

回归结论：

- `blocked`：配置变更后系统不可继续使用，需要回滚或修复。
- `needs_review`：配置可用，但审计或辅助信息不完整。
- `pass`：配置变更闭环通过。

## 11. 配置变更闭环流水线

推荐使用流水线串联校验、补丁生成、可选应用和回归检查：

```bash
python3 tools/run_strategy_config_change_pipeline.py \
  --profile config/investment-profile.example.yaml \
  --changes data/metadata/strategy-config-changes.json \
  --metadata-output data/metadata/strategy-config-change-pipeline.json
```

默认不应用配置，只生成校验、补丁和流水线元数据。

如需应用配置，必须显式传入：

```bash
python3 tools/run_strategy_config_change_pipeline.py \
  --profile config/investment-profile.example.yaml \
  --changes data/metadata/strategy-config-changes.json \
  --apply \
  --applied-by lihongwei \
  --metadata-output data/metadata/strategy-config-change-pipeline.json
```

流水线输出：

- `data/metadata/strategy-config-changes.check.json`
- `reports/strategy-config-patch.md`
- `data/metadata/strategy-config-patch.json`
- `data/metadata/strategy-config-patch.apply.json`，仅 `--apply` 时生成。
- `data/metadata/strategy-config-regression.json`，仅 `--apply` 时生成。
- `data/metadata/strategy-config-change-pipeline.json`

## 12. 生成配置版本快照

配置变更流水线通过后，生成当前投资体系配置快照：

```bash
python3 tools/create_strategy_config_snapshot.py \
  --profile config/investment-profile.example.yaml \
  --pipeline data/metadata/strategy-config-change-pipeline.json \
  --audit data/metadata/strategy-config-patch.apply.json \
  --regression data/metadata/strategy-config-regression.json \
  --output reports/strategy-config-snapshot.md \
  --json-output data/metadata/strategy-config-snapshot.json
```

快照输出：

- `reports/strategy-config-snapshot.md`
- `data/metadata/strategy-config-snapshot.json`

如果 `profile_hash` 变化，就视为新的投资体系配置版本。后续交易计划、执行记录、复盘和回测应尽量引用该快照的 `version_id` 或 `profile_hash`。
