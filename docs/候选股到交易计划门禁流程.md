# 候选股到交易计划门禁流程

本文档定义从候选池行直接生成交易计划、补全人工条件并执行准入门禁的一体化流程。

该流程仍不自动生成买入结论。用户必须显式提供计划买入价、止损价、仓位、止损条件、止盈条件和失效条件。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
config/investment-profile.example.yaml
templates/trade-plan.example.yaml
data/metadata/strategy-health.json
data/metadata/strategy-config-snapshot.json
```

## 2. 执行步骤

`tools/prepare_trade_plan_from_candidate.py` 会按顺序执行：

1. 从候选池读取指定股票。
2. 生成交易计划草稿。
3. 补全用户提供的止损、止盈、失效、观察和复盘字段。
4. 重新计算衍生风控字段。
5. 执行交易计划准入门禁。

## 3. 命令

```bash
python3 tools/prepare_trade_plan_from_candidate.py \
  --candidates data/processed/candidate_pool.csv \
  --code 300750 \
  --name 宁德时代 \
  --exchange SZSE \
  --industry 电力设备 \
  --strategy-config-snapshot data/metadata/strategy-config-snapshot.json \
  --planned-buy-price 200 \
  --stop-loss-price 185 \
  --position-pct 5 \
  --stop-loss-condition "收盘价跌破 185。" \
  --take-profit-condition "达到计划目标区后分批止盈。" \
  --invalidation-condition "趋势或估值证据失效。" \
  --review-focus "候选池证据是否被市场验证。" \
  --strategy-health data/metadata/strategy-health.json \
  --mark-ready
```

输出门禁 JSON：

```bash
python3 tools/prepare_trade_plan_from_candidate.py \
  --code 300750 \
  --planned-buy-price 200 \
  --stop-loss-price 185 \
  --position-pct 5 \
  --stop-loss-condition "收盘价跌破 185。" \
  --take-profit-condition "达到计划目标区后分批止盈。" \
  --invalidation-condition "趋势或估值证据失效。" \
  --gate-output data/metadata/TP-300750.gate.json \
  --json
```

## 4. 输出

默认输出交易计划到：

```text
plans/TP-股票代码.yaml
```

真实交易计划默认不提交到 Git。

生成的交易计划会记录 `strategy_config_snapshot`。如果快照缺失，字段会显式标记为缺失；如果快照存在，后续门禁、执行、持仓和复盘都应沿用同一个 `version_id` 或 `profile_hash`。
