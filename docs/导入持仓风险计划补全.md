# 导入持仓风险计划补全

本文档定义如何为东方财富等外部来源导入的持仓生成风险计划补全草案。

该工具只生成草案，不修改 `positions/*.yaml`，也不产生买卖建议。所有止损价、止盈条件、失效条件和买入理由必须人工确认后再回填。

## 命令

```bash
python3 tools/complete_imported_position_plan.py \
  --positions 'positions/POS-EASTMONEY-*.yaml' \
  --daily-bars data/processed/daily_bars.csv \
  --stop-loss-pct-from-entry 12
```

默认输出：

```text
data/metadata/imported-position-plan-draft.json
reports/imported-position-plan-draft.md
```

## 草案内容

每只持仓会输出：

- 缺失字段：止损价、止盈条件、失效条件、买入理由和关键证据。
- 风险上下文：当前仓位、单票上限、当前价格、买入价和浮盈亏。
- 止损参考：买入价下浮、近20日低点、20日均线和当前价下方参考位。
- 补全建议：止损条件、止盈/降仓条件、失效条件、观察项和下一步。

## 使用边界

- 不直接更新持仓文件。
- 不自动生成退出计划或订单。
- 缺少原始买入逻辑时，草案会默认禁止补仓和做T。
- 仓位超过单票上限时，优先提示降仓复核。
