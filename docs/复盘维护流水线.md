# 复盘维护流水线

本文档定义复盘后的维护流水线。

复盘维护流水线会一次性生成交易复盘分析、复盘冷静期检查、策略健康检查、策略复核任务和流水线元数据，用于支撑策略迭代、配置版本复核和新开仓纪律。

## 1. 输入

默认读取：

```text
config/investment-profile.example.yaml
reviews/*.yaml
data/metadata/candidate-performance.json
```

`candidate-performance.json` 为可选输入。存在时，流水线会把候选池入池后表现带入策略健康报告的 `candidate_observation`，用于观察选股入口质量，但不改变真实交易健康结论。

## 2. 输出

默认输出：

```text
reports/review-analysis.md
reports/strategy-health.md
reports/strategy-review-tasks.md
data/metadata/review-analysis.json
data/metadata/review-cooldown.json
data/metadata/strategy-health.json
data/metadata/strategy-review-tasks.json
data/metadata/review-pipeline.json
```

真实分析结果默认不提交到 Git。

## 3. 命令

```bash
python3 tools/run_review_pipeline.py \
  --reviews reviews/*.yaml \
  --candidate-performance data/metadata/candidate-performance.json
```

输出 JSON：

```bash
python3 tools/run_review_pipeline.py --json
```

## 4. 使用纪律

如果流水线中的冷静期结论为 `cooldown_required`，买入执行会默认阻断。确需例外，必须在执行记录中保留人工确认和例外原因。

如果策略健康结论为 `pause_required` 或 `needs_review`，流水线会同步生成策略复核任务。应先处理对应策略或配置版本的复盘问题，再继续扩大相关交易。
