# 复盘维护流水线

本文档定义复盘后的维护流水线。

复盘维护流水线会一次性生成交易复盘分析、复盘冷静期检查和流水线元数据，用于支撑策略迭代和新开仓纪律。

## 1. 输入

默认读取：

```text
config/investment-profile.example.yaml
reviews/*.yaml
```

## 2. 输出

默认输出：

```text
reports/review-analysis.md
data/metadata/review-analysis.json
data/metadata/review-cooldown.json
data/metadata/review-pipeline.json
```

真实分析结果默认不提交到 Git。

## 3. 命令

```bash
python3 tools/run_review_pipeline.py \
  --reviews reviews/*.yaml
```

输出 JSON：

```bash
python3 tools/run_review_pipeline.py --json
```

## 4. 使用纪律

如果流水线中的冷静期结论为 `cooldown_required`，买入执行会默认阻断。确需例外，必须在执行记录中保留人工确认和例外原因。
