# 东方财富持仓导入

本文档定义从东方财富持仓表导入项目持仓记录的流程。

本流程不接收券商账号密码，不自动登录东方财富，也不自动下单。输入可以是导出的 CSV、复制出来的表格文本，或 macOS 剪贴板中的持仓表。

## 1. 支持字段

`tools/import_eastmoney_positions.py` 会按常见东方财富持仓表头宽松匹配：

| 标准字段 | 常见表头 |
| --- | --- |
| 股票代码 | 证券代码、股票代码、代码 |
| 股票名称 | 证券名称、股票名称、名称 |
| 持仓数量 | 股票余额、证券余额、持仓数量、持有数量 |
| 可用数量 | 可用余额、可用数量、可卖数量 |
| 成本价 | 成本价、持仓成本、买入均价、参考成本价 |
| 当前价 | 当前价、最新价、现价、参考市价 |
| 市值 | 市值、最新市值、股票市值、证券市值、参考市值 |
| 持仓占比 | 持仓占比、仓位占比、持仓比例、市值占比 |
| 盈亏 | 浮动盈亏、持仓盈亏、盈亏、参考盈亏 |
| 盈亏比例 | 盈亏比例、盈亏比、盈亏率、收益率 |

文件编码支持 UTF-8、GB18030、GBK。分隔符支持 CSV 逗号、复制表格常见的 Tab，以及带表头的空格对齐文本。

## 2. 导入命令

### 2.1 找不到导出按钮时：从剪贴板导入

在东方财富持仓表中尝试选中表格行后复制。如果复制成功，可以直接从剪贴板导入：

```bash
python3 tools/import_eastmoney_positions.py \
  --from-clipboard \
  --total-assets 100000 \
  --default-stop-loss-pct 8
```

如果复制出来的是纯文本，也可以先保存到 `data/raw/eastmoney_positions.txt`，再导入：

```bash
python3 tools/import_eastmoney_positions.py \
  --input data/raw/eastmoney_positions.txt \
  --total-assets 100000 \
  --default-stop-loss-pct 8
```

### 2.2 有导出文件时

如果导出文件包含市值，但没有账户总资产，可以用持仓市值合计加现金推算仓位：

```bash
python3 tools/import_eastmoney_positions.py \
  --input data/raw/eastmoney_positions.csv \
  --cash 20000 \
  --default-stop-loss-pct 8
```

如果你知道账户总资产，建议直接传入：

```bash
python3 tools/import_eastmoney_positions.py \
  --input data/raw/eastmoney_positions.csv \
  --total-assets 100000 \
  --default-stop-loss-pct 8
```

输出默认写入：

```text
positions/POS-EASTMONEY-*.yaml
data/metadata/eastmoney-positions.import.json
```

## 3. 后续检查

导入后可自动拉行情并检查做T机会：

```bash
python3 tools/check_t_trade_opportunity.py \
  --position positions/POS-EASTMONEY-示例.yaml \
  --daily-bars data/processed/daily_bars.csv \
  --auto-fetch
```

也可以批量刷新所有导入持仓的日线：

```bash
python3 tools/fetch_daily_bars_sina.py \
  --positions positions/POS-EASTMONEY-*.yaml \
  --output data/processed/daily_bars.csv
```

## 4. 导入后的补充事项

东方财富持仓文件通常只有账户事实数据，不包含买入逻辑、行业、止盈条件和失效条件。导入后仍需要补充：

- 行业。
- 原始买入理由。
- 止盈条件。
- 失效条件。
- 是否允许做T，以及做T失败后的处理方式。

如果不传 `--default-stop-loss-pct`，持仓会缺少止损价，做T检查会保守阻断。
