# 策略健康检查

本文档定义基于复盘分析和冷静期结果的策略健康检查。

策略健康检查不自动修改策略配置，也不预测未来收益。它只把需要暂停新开仓、需要复核或样本不足的策略列出来，供人工调整投资体系。

## 1. 输入

默认读取：

```text
data/metadata/review-analysis.json
data/metadata/review-cooldown.json
```

其中 `review-analysis.json` 的 `discipline.exceptions` 会被纳入策略健康判断，用于识别亏损的纪律例外交易。

## 2. 输出

默认输出：

```text
reports/strategy-health.md
data/metadata/strategy-health.json
```

真实策略健康报告默认不提交到 Git。

## 3. 结论

- `healthy`：没有发现需要暂停或复核的策略。
- `needs_review`：存在胜率、平均收益、组合贡献或亏损纪律例外交易需要复核的策略。
- `pause_required`：存在触发冷静期的策略，应暂停该策略新开仓。

## 4. 命令

```bash
python3 tools/check_strategy_health.py \
  --analysis data/metadata/review-analysis.json \
  --cooldown data/metadata/review-cooldown.json \
  --output reports/strategy-health.md \
  --json-output data/metadata/strategy-health.json
```

可调整阈值：

```bash
python3 tools/check_strategy_health.py \
  --min-trades 5 \
  --min-win-rate-pct 45 \
  --min-avg-return-pct 0
```

## 5. 使用纪律

当结论为 `pause_required` 时，不应继续使用对应策略生成新的买入执行，除非手工记录例外原因并通过冷静期例外流程。

当策略存在亏损纪律例外交易时，策略健康检查会把该策略标记为 `needs_review`。这不等于自动暂停策略，但必须复查破例条件、仓位上限和审批理由，确认破例没有系统性破坏交易纪律。
