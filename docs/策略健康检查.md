# 策略健康检查

本文档定义基于复盘分析和冷静期结果的策略健康检查。

策略健康检查不自动修改策略配置，也不预测未来收益。它只把需要暂停新开仓、需要复核或样本不足的策略，以及需要复核的策略配置版本列出来，供人工调整投资体系。

## 1. 输入

默认读取：

```text
data/metadata/review-analysis.json
data/metadata/review-cooldown.json
data/metadata/candidate-performance.json
```

其中 `review-analysis.json` 的 `discipline.exceptions` 会被纳入策略健康判断，用于识别亏损的纪律例外交易；`by_config_version` 会被纳入配置版本健康判断，用于识别某一版投资体系配置是否表现异常。

`candidate-performance.json` 是可选输入。若存在，策略健康检查会把候选池入池后 5/10/20 日等观察窗口表现按策略汇总到 `candidate_observation`，但不改变真实交易健康状态，也不生成暂停或复核结论。

## 2. 输出

默认输出：

```text
reports/strategy-health.md
data/metadata/strategy-health.json
```

真实策略健康报告默认不提交到 Git。

## 3. 结论

- `healthy`：没有发现需要暂停或复核的策略。
- `needs_review`：存在胜率、平均收益、组合贡献或亏损纪律例外交易需要复核的策略，或存在表现异常的策略配置版本。
- `pause_required`：存在触发冷静期的策略，应暂停该策略新开仓。

## 4. 命令

```bash
python3 tools/check_strategy_health.py \
  --analysis data/metadata/review-analysis.json \
  --cooldown data/metadata/review-cooldown.json \
  --candidate-performance data/metadata/candidate-performance.json \
  --output reports/strategy-health.md \
  --json-output data/metadata/strategy-health.json
```

可调整阈值：

```bash
python3 tools/check_strategy_health.py \
  --min-trades 5 \
  --min-win-rate-pct 45 \
  --min-avg-return-pct 0
```

## 5. 使用纪律

当结论为 `pause_required` 时，不应继续使用对应策略生成新的买入执行，除非手工记录例外原因并通过冷静期例外流程。

当策略存在亏损纪律例外交易时，策略健康检查会把该策略标记为 `needs_review`。这不等于自动暂停策略，但必须复查破例条件、仓位上限和审批理由，确认破例没有系统性破坏交易纪律。

当配置版本胜率、平均收益或组合贡献低于阈值时，策略健康检查会把该配置版本标记为 `needs_review`。这不等于该版本必然失效，但必须确认问题来自配置规则、市场环境，还是策略本身。

候选观察表现只用于判断选股入口质量，例如某策略连续入池后短周期表现偏弱时，应回到因子、行业强度、事件催化和候选池质量门禁中复查。它不能替代真实交易复盘，也不能单独触发买入或卖出。
