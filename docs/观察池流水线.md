# 观察池流水线

本文档定义第一阶段观察池的一键生成流程。

流水线不直接抓取外部数据，只读取已经标准化的本地 CSV。这样可以把“数据获取”和“投资决策生成”分开，便于检查数据来源、复跑和复盘。

## 1. 输入

默认读取：

```text
data/processed/daily_bars.csv
data/processed/financial_metrics.csv
data/processed/valuation_metrics.csv
config/investment-profile.example.yaml
```

可选读取：

```text
data/processed/tradable_universe.csv
```

如果传入可交易股票池，趋势因子只在该股票池内计算。

## 2. 执行步骤

`tools/run_watchlist_pipeline.py` 会按顺序执行：

1. 计算趋势强度基础因子。
2. 筛选趋势强度候选股。
3. 筛选价值质量候选股。
4. 如果传入可交易股票池，计算行业强度因子。
5. 合并多策略候选池，并补充名称、行业、流动性和行业强度。
6. 检查候选池质量。
7. 生成观察池报告。
8. 输出流水线元数据。

趋势因子窗口默认读取投资体系配置中的 `strategies.trend_strength.screening.window`。

## 3. 输出

默认输出：

```text
data/processed/trend_factors.csv
data/processed/trend_candidates.csv
data/processed/value_quality_candidates.csv
data/processed/industry_strength_factors.csv
data/processed/candidate_pool.csv
reports/watchlist.md
data/metadata/watchlist_pipeline.json
```

中间元数据仍由各步骤分别输出到 `data/metadata/`。
候选池质量检查结论写入 `data/metadata/watchlist_pipeline.json` 的 `steps.candidate_pool_check`。

## 4. 命令

```bash
python3 tools/run_watchlist_pipeline.py \
  --daily-bars data/processed/daily_bars.csv \
  --financial-metrics data/processed/financial_metrics.csv \
  --valuation-metrics data/processed/valuation_metrics.csv \
  --universe data/processed/tradable_universe.csv \
  --report-output reports/watchlist.md
```

使用样例数据验证：

```bash
python3 tools/run_watchlist_pipeline.py \
  --daily-bars samples/daily_bars.sample.csv \
  --financial-metrics samples/financial_metrics.sample.csv \
  --valuation-metrics samples/valuation_metrics.sample.csv \
  --trend-factors-output /tmp/trend_factors.csv \
  --trend-factors-metadata /tmp/trend_factors.json \
  --trend-candidates-output /tmp/trend_candidates.csv \
  --trend-candidates-metadata /tmp/trend_candidates.json \
  --value-quality-candidates-output /tmp/value_quality_candidates.csv \
  --value-quality-candidates-metadata /tmp/value_quality_candidates.json \
  --industry-strength-output /tmp/industry_strength_factors.csv \
  --industry-strength-metadata /tmp/industry_strength_factors.json \
  --candidate-pool-output /tmp/candidate_pool.csv \
  --candidate-pool-metadata /tmp/candidate_pool.json \
  --report-output /tmp/watchlist.md \
  --metadata-output /tmp/watchlist_pipeline.json
```

## 5. 决策边界

流水线生成的是观察池，不是交易指令。

进入真实交易前仍必须：

- 补充股票名称、行业和交易假设。
- 明确计划买入价、最大可接受买入价、止损价、仓位和失效条件。
- 生成交易计划。
- 通过风控校验。
