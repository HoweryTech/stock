# 策略复核任务

本文档定义如何从策略健康检查结果生成可追踪的策略复核任务。

该工具不自动修改策略配置，也不自动恢复暂停策略。它只把 `pause_new_entries` 和 `needs_review` 策略转成任务清单，方便人工复核破例规则、胜率、平均收益、组合贡献和仓位纪律。

## 1. 输入

默认读取：

```text
data/metadata/strategy-health.json
```

## 2. 输出

默认输出：

```text
reports/strategy-review-tasks.md
data/metadata/strategy-review-tasks.json
```

真实任务清单默认不提交到 Git。

## 3. 任务来源

会生成任务的策略状态：

- `pause_new_entries`
- `needs_review`

不会生成任务的策略状态：

- `healthy`

## 4. 复核事项

工具会根据策略健康动作自动生成复核事项：

- 连续亏损冷静期：复查最近连续亏损交易，确认是否暂停新开仓。
- 亏损纪律例外：复查破例条件、仓位上限和审批理由。
- 胜率、平均收益或组合贡献偏弱：复查样本与策略假设。
- 样本不足：补充样本，不因样本不足直接扩大仓位。

## 5. 任务状态

新生成的任务默认：

- `task_status: open`
- `resolution: ""`
- `resolved_at: null`
- `history: []`

任务必须通过后续状态更新记录人工结论，不能只在报告外口头处理。

## 6. 命令

```bash
python3 tools/generate_strategy_review_tasks.py \
  --strategy-health data/metadata/strategy-health.json \
  --output reports/strategy-review-tasks.md \
  --json-output data/metadata/strategy-review-tasks.json
```
