# 策略复核任务

本文档定义如何从策略健康检查结果生成可追踪的策略复核任务。

该工具不自动修改策略配置，也不自动恢复暂停策略。它会把 `pause_new_entries` 和 `needs_review` 策略，以及 `needs_review` 配置版本转成任务清单，方便人工复核破例规则、胜率、平均收益、组合贡献、仓位纪律和配置版本表现。

## 1. 输入

默认读取：

```text
data/metadata/strategy-health.json
```

## 2. 输出

默认输出：

```text
reports/strategy-review-tasks.md
data/metadata/strategy-review-tasks.json
```

真实任务清单默认不提交到 Git。

## 3. 任务来源

会生成任务的来源：

- `pause_new_entries`
- 策略 `needs_review`
- 配置版本 `needs_review`

不会生成任务的状态：

- `healthy`

## 4. 复核事项

工具会根据策略健康动作自动生成复核事项：

- 连续亏损冷静期：复查最近连续亏损交易，确认是否暂停新开仓。
- 亏损纪律例外：复查破例条件、仓位上限和审批理由。
- 胜率、平均收益或组合贡献偏弱：复查样本与策略假设。
- 样本不足：补充样本，不因样本不足直接扩大仓位。
- 配置版本表现偏弱：复查问题来自配置规则、市场阶段，还是具体策略；形成保留、回滚、拆分适用场景或调整规则的结论。

## 5. 任务状态

新生成的任务默认：

- `task_status: open`
- `resolution: ""`
- `resolved_at: null`
- `history: []`

任务必须通过后续状态更新记录人工结论，不能只在报告外口头处理。

## 6. 生成任务命令

```bash
python3 tools/generate_strategy_review_tasks.py \
  --strategy-health data/metadata/strategy-health.json \
  --output reports/strategy-review-tasks.md \
  --json-output data/metadata/strategy-review-tasks.json
```

## 7. 更新任务状态

支持的任务处理状态：

- `open`：仍待处理。
- `resolved`：已形成结论。
- `deferred`：暂缓处理，但必须说明原因。

`resolved` 和 `deferred` 必须填写 `--resolution`。工具会追加 `history`，并更新任务统计字段。

```bash
python3 tools/update_strategy_review_task.py \
  --tasks data/metadata/strategy-review-tasks.json \
  --task-id STRATEGY-REVIEW-TREND-STRENGTH-NEEDS-REVIEW \
  --status resolved \
  --resolution "维持策略，但降低例外仓位上限到 2%。" \
  --updated-by lihongwei
```
