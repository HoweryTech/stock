# 交易计划准入门禁

本文档定义交易计划从草稿进入人工确认前的统一门禁。

门禁由三层组成：

1. 草稿质量检查：计划是否写完整，是否仍有占位内容，是否能追溯到候选池证据。
2. 风控校验：计划是否违反价格、仓位、止损、交易状态等风险边界。
3. 策略健康检查：计划策略是否已经被复盘结果标记为暂停新开仓或需要复核。

## 1. 默认行为

`tools/check_trade_plan_gate.py` 默认先执行草稿质量检查。

如果草稿质量检查为 `blocked`，则不会继续执行风控校验，因为此时风控结果容易误导用户，以为“只是风控通过/不通过”，而实际计划本身还没写完整。

## 2. 结论

门禁结论包括：

- `pass`：草稿质量检查和风控校验都通过。
- `needs_confirmation`：没有阻断项，但存在提醒项或需要人工确认。
- `blocked_by_quality`：草稿质量检查阻断。
- `blocked_by_risk`：草稿质量通过，但风控校验阻断。
- `blocked_by_strategy_health`：计划策略已被策略健康检查标记为暂停新开仓。

## 3. 命令

```bash
python3 tools/check_trade_plan_gate.py \
  --profile config/investment-profile.example.yaml \
  --plan plans/TP-示例.yaml \
  --strategy-health data/metadata/strategy-health.json
```

输出 JSON：

```bash
python3 tools/check_trade_plan_gate.py \
  --profile config/investment-profile.example.yaml \
  --plan plans/TP-示例.yaml \
  --json
```

如需调试，可以强制在草稿质量阻断时仍执行风控：

```bash
python3 tools/check_trade_plan_gate.py \
  --plan plans/TP-示例.yaml \
  --run-risk-even-if-quality-blocked
```

## 4. 推荐顺序

```text
候选池质量检查
  -> 候选股生成交易计划草稿
  -> 交易计划准入门禁
  -> 人工确认
  -> 交易执行记录
```
