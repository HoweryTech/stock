# 日常检查流水线

本文档定义如何一键生成执行闭环总检查和每日操作摘要。

该流水线不产生买卖建议，只按固定顺序生成审计材料，避免每日摘要读取旧的执行闭环状态。

流水线生成的 `reports/` 和 `data/metadata/` 文件属于本地运行产物，默认不提交到 Git。提交边界见：[运行产物与 Git 边界](./运行产物与Git边界.md)。

## 1. 输出

默认生成：

```text
reports/execution-loop-check.md
data/metadata/execution-loop-check.json
reports/daily-summary.md
data/metadata/daily-summary.json
data/metadata/daily-check-pipeline.json
```

## 2. 执行顺序

1. 运行执行闭环总检查，检查买入执行、持仓、卖出执行和复盘之间的下游记录是否完整，是否存在孤儿记录。
2. 写入 `data/metadata/execution-loop-check.json`。
3. 生成每日操作摘要，并读取刚生成的执行闭环检查结果。
4. 如果已有 `data/metadata/holding-action-draft.json`，每日摘要会展示持仓趋势状态和价格动作矩阵；流水线本身不自动生成该草案。
5. 写入流水线元数据，记录每一步的输出和结论。

## 3. 命令

```bash
python3 tools/run_daily_check_pipeline.py
```

输出 JSON：

```bash
python3 tools/run_daily_check_pipeline.py --json
```

自动化归档场景下，如果希望闭环 `blocked` 时仍返回成功退出码，但在元数据中保留阻断结论：

```bash
python3 tools/run_daily_check_pipeline.py --continue-on-blocked
```

命令退出码规则：

- `0`：执行闭环总检查不是 `blocked`，或已显式传入 `--continue-on-blocked`。
- `1`：执行闭环总检查为 `blocked`，且未传入 `--continue-on-blocked`。
- `2`：读取或生成过程异常。
