# 候选池质量检查

本文档定义候选股进入交易计划前的证据门禁检查。

候选池质量检查不改变候选排序，也不输出买入建议。它只回答一个问题：候选股是否具备进入交易计划草稿的最低证据完整性和上下文安全性。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
```

该文件由 `tools/merge_candidate_pool.py` 或 `tools/run_watchlist_pipeline.py` 生成。

可选读取：

```text
data/processed/tradable_universe.csv
```

如果提供可交易股票池，候选股必须仍在该股票池中，否则视为阻断。

## 2. 检查项

阻断项：

- 缺少股票代码。
- 缺少策略来源。
- 缺少入选证据。
- 缺少主策略。
- 多策略候选的策略来源少于 2 个。
- 趋势候选缺少交易日或趋势分。
- 价值质量候选缺少报告期或价值质量分。
- 事件催化候选缺少事件日期或事件分。
- 传入可交易股票池时，候选股不在可交易股票池中。

提醒项：

- 缺少综合排序分、策略共振、数据质量或风险扣分等评分解释字段。
- 数据质量状态为 `weak`。
- 风险扣分过高。
- 流动性评分偏低。
- 缺少显式风险提示。
- 单策略候选需要补齐其他维度证据。
- 候选证据少于 2 个非风险维度。
- 缺少流动性证据。
- 价值质量候选未看到估值分位证据。
- 传入检查日时，趋势交易日过旧。

## 3. 结论

检查结论包括：

- `pass`：没有阻断项和提醒项。
- `needs_review`：没有阻断项，但存在提醒项。
- `blocked`：存在阻断项。

`blocked` 不应进入交易计划草稿。

## 4. 命令

```bash
python3 tools/check_candidate_pool.py \
  --candidates data/processed/candidate_pool.csv
```

带可交易股票池和时效检查：

```bash
python3 tools/check_candidate_pool.py \
  --candidates data/processed/candidate_pool.csv \
  --tradable-universe data/processed/tradable_universe.csv \
  --as-of 2026-07-22 \
  --max-trend-age-days 5
```

输出 JSON：

```bash
python3 tools/check_candidate_pool.py \
  --candidates data/processed/candidate_pool.csv \
  --json
```
