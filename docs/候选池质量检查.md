# 候选池质量检查

本文档定义候选股进入交易计划前的字段完整性检查。

候选池质量检查不改变候选排序，也不输出买入建议。它只回答一个问题：候选股是否具备进入交易计划草稿的最低证据完整性。

## 1. 输入

默认读取：

```text
data/processed/candidate_pool.csv
```

该文件由 `tools/merge_candidate_pool.py` 或 `tools/run_watchlist_pipeline.py` 生成。

## 2. 检查项

阻断项：

- 缺少股票代码。
- 缺少策略来源。
- 缺少入选证据。
- 缺少主策略。
- 多策略候选的策略来源少于 2 个。
- 趋势候选缺少交易日或趋势分。
- 价值质量候选缺少报告期或价值质量分。

提醒项：

- 缺少显式风险提示。
- 单策略候选需要补齐其他维度证据。
- 价值质量候选未看到估值分位证据。

## 3. 结论

检查结论包括：

- `pass`：没有阻断项和提醒项。
- `needs_review`：没有阻断项，但存在提醒项。
- `blocked`：存在阻断项。

`blocked` 不应进入交易计划草稿。

## 4. 命令

```bash
python3 tools/check_candidate_pool.py \
  --candidates data/processed/candidate_pool.csv
```

输出 JSON：

```bash
python3 tools/check_candidate_pool.py \
  --candidates data/processed/candidate_pool.csv \
  --json
```
