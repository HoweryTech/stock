# 估值指标导入

本文档定义第一阶段估值指标 CSV 的标准结构和导入规则。

当前阶段先使用本地 CSV 作为入口。后续接入 AKShare、Tushare、BaoStock 或其他数据源时，也应先转换为本文档定义的标准结构，再进入价值质量策略、风险过滤和候选股解释模块。

## 1. 目录约定

```text
data/
  raw/          原始估值指标文件
  processed/    标准化后的估值指标
  metadata/     导入元数据和质量报告
```

真实估值数据默认不提交到 Git。

## 2. 估值指标 CSV

第一版估值指标文件至少包含以下字段：

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| trade_date | 是 | date | 交易日，格式 YYYY-MM-DD |
| code | 是 | string | 股票代码，保留前导 0 |
| pe_ttm | 否 | number | 滚动市盈率 |
| pb | 否 | number | 市净率 |
| ps_ttm | 否 | number | 滚动市销率 |
| dividend_yield | 否 | number | 股息率，百分比 |
| market_cap | 否 | number | 总市值，元 |
| pe_percentile | 否 | number | PE 历史分位，0-100 |
| pb_percentile | 否 | number | PB 历史分位，0-100 |
| data_source | 否 | string | 数据来源 |
| updated_at | 否 | date | 数据更新时间，格式 YYYY-MM-DD |

## 3. 导入命令

```bash
python3 tools/import_valuation_metrics.py \
  --input samples/valuation_metrics.sample.csv \
  --output data/processed/valuation_metrics.csv \
  --metadata-output data/metadata/valuation_metrics.import.json
```

导入工具会执行：

- 必填字段检查。
- 交易日校验。
- 股票代码格式校验。
- 数值字段标准化。
- 估值分位 0-100 范围校验。
- 同一股票同一交易日重复记录检查。
- 标准 CSV 输出。
- 导入元数据输出。

## 4. 输出标准

标准输出 CSV 字段顺序固定为：

```text
trade_date,code,pe_ttm,pb,ps_ttm,dividend_yield,market_cap,pe_percentile,pb_percentile,data_source,updated_at
```

后续价值质量策略会读取 `data/processed/valuation_metrics.csv`，用于补充估值约束和估值风险提示。
