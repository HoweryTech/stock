# 交易计划草稿质量检查

本文档定义交易计划进入风控校验前的草稿质量门禁。

`risk_check.py` 负责判断交易计划是否违反仓位、价格、止损和交易状态等风控规则。`check_trade_plan_quality.py` 负责更早一步：判断草稿是否写完整，是否还存在占位内容，是否能追溯到候选池证据。

## 1. 输入

默认读取真实交易计划 YAML：

```text
plans/*.yaml
```

真实交易计划默认不提交到 Git。

## 2. 阻断项

触发阻断时，不应进入风控校验：

- 股票代码、股票名称、交易所、行业、策略来源、买入理由缺失或仍为占位内容。
- 计划买入价、止损价、计划仓位缺失。
- 止损条件、止盈条件、失效条件缺失。
- 止损价不低于计划买入价。
- 单笔最大亏损计算缺失。
- 关键证据少于 2 条。
- 反证和风险少于 1 条。
- 关键证据或风险仍包含占位内容。

## 3. 提醒项

触发提醒时，可以继续完善，但不应直接交易：

- 未看到观察池来源或策略证据前缀。
- 价值质量计划未看到估值或估值分位证据。
- 单笔最大亏损声明值与按买入价、止损价、仓位计算值不一致。

## 4. 信息项

- 复盘关注点为空时，建议补充，便于退出后归因。

## 5. 命令

```bash
python3 tools/check_trade_plan_quality.py \
  --plan plans/TP-示例.yaml
```

输出 JSON：

```bash
python3 tools/check_trade_plan_quality.py \
  --plan plans/TP-示例.yaml \
  --json
```

## 6. 推荐顺序

```text
候选池
  -> 交易计划草稿
  -> 草稿质量检查
  -> 风控校验
  -> 人工确认
```
