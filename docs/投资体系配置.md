# 投资体系配置

本文档用于固化个人投资体系、交易边界和风控纪律。

后续系统中的选股、交易计划校验、持仓风险跟踪和复盘归因，都应优先读取这套规则，而不是依赖临时判断。

对应的机器可读配置文件为：

- `config/investment-profile.example.yaml`

## 1. 配置目标

投资体系配置要解决三个问题：

1. 哪些机会符合我的体系。
2. 哪些交易即使看起来有机会也不能做。
3. 当判断错误时，系统如何强制我控制损失并留下复盘证据。

## 2. 基础投资画像

第一版先按默认画像建模，后续根据真实偏好调整。

```text
市场范围：A 股
主要用途：选股、交易计划、风控、复盘
交易方式：人工确认后交易
默认周期：波段 + 中线
默认风格：价值质量 + 趋势强度 + 事件催化
核心约束：先控制风险，再追求收益
```

## 3. 风控参数

第一版建议使用偏保守参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 单笔最大亏损 | 1% 总资产 | 买入前按止损价计算最大亏损 |
| 单票最大仓位 | 10% 总资产 | 防止单一股票风险过高 |
| 单行业最大仓位 | 25% 总资产 | 防止行业集中暴露 |
| 总仓位上限 | 80% 总资产 | 正常市场环境下保留现金 |
| 弱势市场总仓位上限 | 30% 总资产 | 大盘环境不利时降低暴露 |
| 连续亏损冷静期 | 3 笔亏损后暂停新开仓 | 防止情绪化交易 |
| 追高阈值 | 超过计划买入价 3% 拦截 | 防止临时追涨 |

这些参数不是永久规则，而是第一版安全默认值。真实使用一段时间后，应通过复盘结果调整。

## 4. 禁买规则

以下情况默认禁止买入：

- ST 或存在明确退市风险。
- 当前停牌或交易状态异常。
- 长期成交额过低，流动性不足。
- 近期出现重大监管、财务、诚信风险。
- 财报质量明显异常，例如利润增长但经营现金流长期恶化。
- 股价已经明显偏离计划买入价。
- 没有明确止损价。
- 没有明确买入理由。
- 买入理由无法对应到任何已定义策略。
- 当前组合仓位、单票仓位或行业仓位超过上限。
- 连续亏损触发冷静期。

## 5. 允许策略

第一版允许三类策略。

### 5.1 价值质量策略

适合中线观察。

必须关注：

- ROE
- 经营现金流
- 资产负债率
- 毛利率
- 净利率
- 营收同比
- 扣非净利润同比
- 估值分位

候选股应满足：

- 财务质量稳定。
- 估值不处于明显极端高位。
- 现金流与利润质量匹配。
- 没有重大财务风险信号。

### 5.2 趋势强度策略

适合波段交易。

必须关注：

- 均线结构
- 相对指数强度
- 成交额变化
- 突破、回踩、支撑和压力
- 行业强度
- 波动率

候选股应满足：

- 趋势强于市场或所属行业。
- 成交额支持趋势变化。
- 买点不能明显追高。
- 止损位置清晰。

### 5.3 事件催化策略

适合观察市场预期变化。

必须关注：

- 业绩预告
- 回购
- 增持
- 政策催化
- 并购重组
- 订单或产品进展
- 监管问询和风险公告

候选股应满足：

- 事件真实可验证。
- 事件能影响盈利预期、估值预期或资金关注度。
- 风险公告和反证被同时检查。

## 6. 买入前强制检查

每次买入前，系统必须要求生成交易计划。

必填字段：

```text
标的代码
标的名称
策略来源
买入理由
关键证据
反证和风险
计划买入价
最大可接受买入价
止损价
计划仓位
单笔最大亏损
卖出条件
失效条件
```

校验规则：

- 无策略来源，禁止交易。
- 无止损价，禁止交易。
- 单笔最大亏损超过阈值，禁止交易。
- 计划仓位超过单票上限，禁止交易。
- 行业仓位超过上限，禁止交易。
- 当前价格超过最大可接受买入价，禁止交易。
- 存在未处理重大风险，禁止交易或强提醒。

## 7. 持仓期间监控

持仓期间系统需要持续检查：

- 是否触发止损。
- 是否触发止盈或移动止盈。
- 是否跌破趋势失效条件。
- 是否出现公告风险。
- 是否出现财报风险。
- 是否行业逻辑变化。
- 是否组合集中度过高。
- 是否持仓理由已经失效。

持仓状态建议分为：

```text
正常持有
接近止损
触发止损
达到止盈观察区
逻辑待验证
持仓理由失效
需要复盘
```

## 8. 卖出纪律

第一版卖出条件包括：

- 触发止损。
- 趋势结构破坏。
- 基本面假设被证伪。
- 事件催化落空。
- 达到计划止盈条件。
- 出现更高优先级风险。
- 组合需要降仓。
- 交易进入复盘期，暂不允许继续加仓。

卖出后必须进入复盘流程。

## 9. 复盘归因

每笔完成交易都必须分类：

```text
策略内盈利
策略内亏损
执行错误盈利
执行错误亏损
随机波动
规则设计缺陷
数据或信息缺失
```

常见错误标签：

- 追高
- 重仓
- 无计划交易
- 止损慢
- 过早止盈
- 忽略公告风险
- 忽略市场环境
- 买入理由不充分
- 交易理由与策略不匹配
- 情绪化交易

## 10. 后续需要用户确认

以下参数需要后续根据你的真实投资偏好确认：

- 最大可接受年度回撤。
- 单笔最大亏损是否采用 0.5%、1% 还是其他值。
- 单票最大仓位是否采用 5%、10% 还是其他值。
- 是否允许满仓。
- 是否接受短线交易。
- 是否接受创业板、科创板、北交所。
- 是否有明确禁买行业。
- 是否有已有交易记录可以导入。
- 是否已有偏好的技术指标或基本面指标。
